# 4.1.1 Введение в Vue 3

Vue — это фреймворк для построения интерфейсов. Вместо того чтобы вручную обновлять DOM через `document.querySelector`, ты описываешь, как интерфейс должен выглядеть при тех или иных данных — а Vue сам решает, что и когда перерисовать.

## 📖 Зачем Vue, если есть vanilla JavaScript?

На ванильном JavaScript интерфейс обновляется вручную: ты сам находишь нужный элемент и меняешь его содержимое.

```javascript
// Ванильный JS — каждое обновление UI прописываешь руками
function renderWeather(data) {
  document.querySelector(".city").textContent = data.city;
  document.querySelector(".temp").textContent = `${data.temperature}°C`;
  document.querySelector(".humidity").textContent = `${data.humidity}%`;
}

// Пользователь добавил город в избранное?
function toggleFavorite(city) {
  const btn = document.querySelector(`[data-city="${city}"] .fav-btn`);
  btn.classList.toggle("active");
  // Ещё нужно обновить список избранного в сайдбаре...
  // И счётчик в шапке...
  // И сохранить в localStorage...
  // Не забыть синхронизировать с другими карточками, если город там тоже есть
}
```

Когда приложение маленькое — это терпимо. Но когда экранов становится больше, данные используются в нескольких местах одновременно и состояние усложняется, ручное управление DOM превращается в источник багов: легко забыть обновить один из элементов или обновить их в неправильном порядке.

**Корень проблемы** — данные и их отображение живут отдельно и не знают друг о друге. Ты сам отвечаешь за их синхронизацию.

Vue решает это через реактивность: ты объявляешь данные и **описываешь** в шаблоне, как они должны выглядеть. Когда данные меняются — Vue сам обновляет всё, что их использует, где бы они ни отображались:

```vue
<script setup>
import { ref } from "vue";

const city = ref("Москва");
const temperature = ref(20);
const isFavorite = ref(false);

// Изменяем данные — Vue обновит шаблон автоматически
function toggleFavorite() {
  isFavorite.value = !isFavorite.value;
}
</script>

<template>
  <div class="weather-card">
    <h2>{{ city }}</h2>
    <p>{{ temperature }}°C</p>
    <!-- Кнопка обновится сама, как только isFavorite изменится -->
    <button @click="toggleFavorite">
      {{ isFavorite ? "★ В избранном" : "☆ Добавить" }}
    </button>
  </div>
</template>
```

Кроме реактивности Vue даёт компонентный подход: интерфейс разбивается на самостоятельные переиспользуемые части — каждая со своей логикой, шаблоном и стилями в одном файле.

## 📖 Single File Component (SFC)

Основная единица Vue-приложения — **компонент**. В Vue 3 компоненты пишутся в `.vue` файлах, которые называются Single File Component (SFC). Каждый файл состоит из трёх блоков:

```vue
<!-- WeatherCard.vue -->

<!-- 1. Шаблон — HTML с синтаксисом Vue -->
<template>
  <div class="weather-card">
    <h2>{{ city }}</h2>
    <p>{{ temperature }}°C</p>
  </div>
</template>

<!-- 2. Логика — TypeScript с Composition API -->
<script setup lang="ts">
const city = "Москва";
const temperature = 20;
</script>

<!-- 3. Стили — CSS, опционально scoped -->
<style scoped>
.weather-card {
  padding: 16px;
  border-radius: 8px;
}
</style>
```

**Три блока в одном файле** — это удобно: всё, что относится к одному компоненту, находится рядом.

### `<script setup lang="ts">`

`setup` — это современный синтаксис Composition API. Всё, что объявлено внутри, автоматически доступно в шаблоне. `lang="ts"` включает TypeScript.

```vue
<script setup lang="ts">
// Это доступно в шаблоне автоматически:
const city = "Москва";
const temperature = 20;

function refresh() {
  console.log("обновляю...");
}
</script>

<template>
  <!-- city, temperature и refresh — все доступны здесь -->
  <button @click="refresh">{{ city }}: {{ temperature }}°C</button>
</template>
```

### `scoped` стили

Атрибут `scoped` означает, что стили применяются только к элементам **этого** компонента и не утекают наружу:

```vue
<style scoped>
/* Этот стиль работает только внутри WeatherCard.vue */
.weather-card { background: white; }
</style>
```

## 📖 Reactivity в двух словах

Главная особенность Vue — **реактивность**. Когда реактивные данные меняются, Vue автоматически обновляет все части шаблона, которые их используют.

```vue
<script setup lang="ts">
import { ref } from "vue";

// ref() делает переменную реактивной
const temperature = ref(20);

// Через секунду обновим значение — Vue сам обновит шаблон
setTimeout(() => {
  temperature.value = 25; // .value — так читаем и пишем ref
}, 1000);
</script>

<template>
  <!-- Это обновится автоматически, когда temperature.value изменится -->
  <p>{{ temperature }}°C</p>
</template>
```

Подробно про `ref`, `reactive` и `computed` — в следующем разделе 4.1.2. Пока главное понять принцип: **данные изменились → шаблон обновился сам**.

## 🔧 Практика 4.1.1.1: Создание Vue 3 проекта

Создадим новый Vue 3 проект с Vite и TypeScript через официальный инструмент.

**Шаг 1: Запусти мастер создания проекта**

```bash
npm create vue@latest
```

Мастер задаст несколько вопросов. Выбери вот так:

```
✔ Project name: weather-vue
✔ Add TypeScript? › Yes
✔ Add JSX Support? › No
✔ Add Vue Router for Single Page Application development? › No
✔ Add Pinia for state management? › No
✔ Add Vitest for Unit Testing? › No
✔ Add an End-to-End Testing Solution? › No
✔ Add ESLint for code quality? › Yes
✔ Add Prettier for code formatting? › Yes
```

> 💡 Router и Pinia пока отключаем — добавим в разделах 4.2.3 и 4.2.4, когда до них дойдём.

**Шаг 2: Установи зависимости и запусти проект**

```bash
cd weather-vue
npm install
npm run dev
```

Открой браузер на `http://localhost:5173` — увидишь стартовую страницу Vue.

**Шаг 3: Изучи структуру проекта**

```
weather-vue/
├── public/              # Статические файлы (favicon и т.д.)
├── src/
│   ├── assets/          # Изображения, шрифты, глобальные стили
│   ├── components/      # Vue-компоненты
│   │   └── HelloWorld.vue
│   ├── App.vue          # Корневой компонент
│   └── main.ts          # Точка входа — монтирует Vue в index.html
├── index.html           # HTML-шаблон
├── tsconfig.json        # Конфиг TypeScript
├── vite.config.ts       # Конфиг Vite
└── package.json
```

**Шаг 4: Посмотри на точку входа**

Открой `src/main.ts`:

```typescript
import { createApp } from "vue";
import App from "./App.vue";

import "./assets/main.css";

// Создаём Vue-приложение и монтируем его в <div id="app"> из index.html
createApp(App).mount("#app");
```

**Шаг 5: Посмотри на корневой компонент**

Открой `src/App.vue` — это корневой компонент, который рендерится первым. Сейчас там содержимое от шаблона. Удали его и замени на минимальный вариант:

```vue
<template>
  <main>
    <h1>Погода</h1>
  </main>
</template>

<script setup lang="ts">
// Пока пусто — скоро добавим компоненты
</script>

<style scoped>
main {
  max-width: 600px;
  margin: 0 auto;
  padding: 24px;
}
</style>
```

Сохрани — браузер автоматически обновится (HMR работает из коробки).

## 🔧 Практика 4.1.1.2: Первый компонент WeatherCard.vue

Создадим компонент для отображения данных о погоде.

**Шаг 1: Создай файл компонента**

Создай `src/components/WeatherCard.vue`:

```vue
<template>
  <div class="weather-card">
    <div class="weather-card__header">
      <h2 class="weather-card__city">{{ city }}</h2>
      <span class="weather-card__country">{{ country }}</span>
    </div>

    <div class="weather-card__body">
      <p class="weather-card__temp">{{ temperature }}°C</p>
      <p class="weather-card__description">{{ description }}</p>
    </div>

    <div class="weather-card__footer">
      <span>💧 {{ humidity }}%</span>
      <span>🌡 Ощущается как {{ feelsLike }}°C</span>
    </div>
  </div>
</template>

<script setup lang="ts">
// Пока используем статические данные — реактивность добавим в 4.1.2
const city = "Москва";
const country = "RU";
const temperature = 20;
const description = "Облачно";
const humidity = 65;
const feelsLike = 18;
</script>

<style scoped>
.weather-card {
  background: white;
  border-radius: 12px;
  padding: 24px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

.weather-card__header {
  display: flex;
  align-items: baseline;
  gap: 8px;
  margin-bottom: 16px;
}

.weather-card__city {
  font-size: 24px;
  margin: 0;
}

.weather-card__country {
  color: #666;
  font-size: 14px;
}

.weather-card__temp {
  font-size: 48px;
  font-weight: bold;
  margin: 0 0 8px;
}

.weather-card__description {
  color: #555;
  margin: 0 0 16px;
  text-transform: capitalize;
}

.weather-card__footer {
  display: flex;
  gap: 16px;
  color: #666;
  font-size: 14px;
}
</style>
```

**Шаг 2: Подключи компонент в App.vue**

```vue
<template>
  <main>
    <h1>Погода</h1>
    <WeatherCard />
  </main>
</template>

<script setup lang="ts">
import WeatherCard from "./components/WeatherCard.vue";
</script>

<style scoped>
main {
  max-width: 600px;
  margin: 0 auto;
  padding: 24px;
}
</style>
```

Сохрани — в браузере появится карточка со статическими данными.

**Шаг 3: Посмотри, как Vue компилирует шаблон**

Открой DevTools → Elements. Найди `.weather-card` — увидишь, что Vue скомпилировал `<template>` в обычный HTML. Атрибут `data-v-xxxxxxxx` на элементах — это маркер для `scoped` стилей.

## 🔧 Практика 4.1.1.3: Перенос TypeScript-типов

Типы `WeatherData` из TypeScript-раздела перенесём в новый проект. Это и есть практическая польза от изученного TypeScript — типы не привязаны к архитектуре.

**Шаг 1: Создай папку для типов**

```bash
mkdir src/types
```

**Шаг 2: Создай `src/types/weather.ts`**

Это та же структура, что была в TypeScript-проекте — переносим как есть:

```typescript
// Базовые данные о погоде — приходят из OpenWeatherMap API
export interface WeatherData {
  city: string;
  country: string;
  temperature: number;
  feelsLike: number;
  humidity: number;
  description: string;
  icon: string;
  timestamp: number;
}

// Статус загрузки — используем для управления UI
export type LoadingStatus = "idle" | "loading" | "success" | "error";

// Единицы измерения температуры
export type TemperatureUnit = "metric" | "imperial";
```

**Шаг 3: Используй типы в компоненте**

Обнови `src/components/WeatherCard.vue` — добавь типизацию:

```vue
<script setup lang="ts">
import type { WeatherData } from "../types/weather";

// Статические данные для проверки — типизированный объект
const weatherData: WeatherData = {
  city: "Москва",
  country: "RU",
  temperature: 20,
  feelsLike: 18,
  humidity: 65,
  description: "Облачно",
  icon: "04d",
  timestamp: Date.now(),
};
</script>

<template>
  <div class="weather-card">
    <div class="weather-card__header">
      <h2 class="weather-card__city">{{ weatherData.city }}</h2>
      <span class="weather-card__country">{{ weatherData.country }}</span>
    </div>

    <div class="weather-card__body">
      <p class="weather-card__temp">{{ weatherData.temperature }}°C</p>
      <p class="weather-card__description">{{ weatherData.description }}</p>
    </div>

    <div class="weather-card__footer">
      <span>💧 {{ weatherData.humidity }}%</span>
      <span>🌡 Ощущается как {{ weatherData.feelsLike }}°C</span>
    </div>
  </div>
</template>
```

**Шаг 4: Проверь TypeScript**

```bash
npm run type-check
```

Команда запускает `vue-tsc --noEmit` — проверяет типы в `.vue` файлах. Ошибок быть не должно.

**Задача:** Расширь `WeatherCard.vue` и добавь второй компонент `LoadingSpinner.vue`.

### Требования:

