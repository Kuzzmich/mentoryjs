# 🟢 4. Vue.js

Vue.js — прогрессивный фреймворк для построения пользовательских интерфейсов. Здесь изучаем Vue 3 с Composition API и TypeScript — именно тот стек, который используется в современных проектах.

---

Здесь ты научишься:
- строить реактивные интерфейсы на компонентах,
- организовывать переиспользуемую логику через composables,
- настраивать роутинг и управление состоянием,
- типизировать Vue-компоненты через TypeScript,
- тестировать компоненты и оптимизировать рендеринг.

---

> 💡 **Совет:** Весь раздел написан на **Composition API** с синтаксисом `<script setup>` — это стандарт Vue 3. Options API упоминается только как справка: ты встретишь его в старых проектах, но писать новое лучше через Composition API.

---

## 🌤 Сквозной проект: Погодное приложение на Vue 3

Создаём погодное приложение заново — на этот раз как полноценное Vue SPA. Домен тот же (погода, города, OpenWeatherMap), но архитектура другая: компоненты, composables и стор вместо классов и декораторов.

**Почему новый проект, а не конвертация?** TypeScript-приложение из раздела 3 строилось для демонстрации возможностей системы типов: декораторы, mapped types, module augmentation. Vue строится иначе — реактивные данные, компонентное дерево, composables. Попытка «конвертировать» один в другой мешала бы учиться Vue.

**Что переносится из TS-проекта:**
- TypeScript-типы (`WeatherData`, `WeatherApiResponse`) — копируем без изменений, это и есть практическая польза от раздела TypeScript
- Логика обращения к API и API-ключ — те же endpoint'ы OpenWeatherMap
- Доменные знания — студент уже знает задачу, не нужно объяснять контекст

Каждый раздел добавляет новый слой: от первого компонента до полноценного SPA с роутингом и стором.

### Этап 1: Основы (4.1)

**4.1.1 Введение в Vue 3**
- Что такое SFC (Single File Component): `<template>`, `<script setup>`, `<style>`
- Создание нового проекта через `npm create vue@latest` (Vite + TypeScript)
- Подключение TypeScript-типов из раздела 3 в новый проект
- Первый компонент: `WeatherCard.vue`

**4.1.2 Реактивность**
- `ref` для примитивов, `reactive` для объектов
- `computed` — вычисляемые значения
- Разница между `ref` и `reactive` на практике

**4.1.3 Template и директивы**
- Интерполяция `{{ }}` и атрибуты `:bind`
- `v-if` / `v-else` / `v-show` — условный рендеринг
- `v-for` — списки с `:key`
- `v-model` — двустороннее связывание
- `v-on` / `@click` — обработка событий

**4.1.4 Компоненты: props и emit**
- `defineProps<T>()` с TypeScript-интерфейсом
- `defineEmits<T>()` с типизированными событиями
- Однонаправленный поток данных

**4.1.5 Lifecycle hooks**
- `onMounted`, `onUnmounted`, `onUpdated`
- Когда что использовать
- Загрузка данных в `onMounted`

**4.1.6 Watchers**
- `watch` — реакция на изменение конкретного значения
- `watchEffect` — автоматическое отслеживание зависимостей
- Когда `watch`, когда `computed`, когда `watchEffect`

### Этап 2: Продвинутый уровень (4.2)

**4.2.1 Composables**
- Что такое composable и зачем он нужен
- `useWeather()`, `useLocalStorage()`, `useDebounce()`
- Правила composables: префикс `use`, вызов только в `setup`

**4.2.2 Слоты и динамические компоненты**
- Слоты по умолчанию и именованные слоты
- Scoped slots
- `<component :is="...">` — динамический рендеринг

**4.2.3 Vue Router**
- Установка и настройка роутера
- Динамические маршруты: `/weather/:city`
- `useRoute`, `useRouter` — навигация из компонента
- Навигационные гарды

**4.2.4 Pinia**
- Зачем нужен стор, когда `ref` уже не хватает
- `defineStore` — создание стора
- `storeToRefs` — деструктуризация без потери реактивности
- Типизация стора с TypeScript

**4.2.5 Vue + TypeScript углублённо**
- Типизация `ref` и `reactive`
- `defineComponent` и `PropType` для сложных случаев
- Типизация `provide` / `inject`
- Шаблонные ссылки: `ref<HTMLElement | null>`

### Этап 3: Экспертный уровень (4.3)

**4.3.1 Оптимизация рендеринга**
- Как Vue определяет, что перерисовывать
- `shallowRef`, `shallowReactive`, `markRaw`
- `v-memo` — мемоизация части шаблона
- `defineAsyncComponent` — ленивая загрузка

**4.3.2 Тестирование**
- Vitest + Vue Test Utils
- Тестирование props, emit, slots
- Мокирование composables
- Тестирование Pinia-сторов

**4.3.3 SSR с Nuxt** *(опционально)*
- Что такое SSR и зачем он нужен
- Структура Nuxt-проекта
- File-based routing
- `useFetch` и `useAsyncData`
