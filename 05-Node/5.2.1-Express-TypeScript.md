# 5.2.1 Express + TypeScript: инициализация проекта

Express — минималистичный фреймворк для Node.js. Он не навязывает структуру и не решает за тебя, как организовать код. В этом его сила и одновременно ловушка: без чёткой структуры проект быстро превращается в один большой файл. Здесь разберём, как стартовать правильно с TypeScript с самого начала.

---

## 🎯 Что ты узнаешь

- Как установить Express и подключить TypeScript-типы
- Как типизировать `Request`, `Response`, `NextFunction`
- Как создать первый маршрут с типизированными параметрами
- Как сделать прокси к внешнему API, не раскрывая ключ клиенту

## 📖 Установка и минимальный сервер

```bash
npm install express
npm install -D @types/express
```

`@types/express` — пакет с TypeScript-описаниями типов для Express. Без него TypeScript не будет знать, что такое `Request`, `Response` и другие объекты Express.

Минимальный сервер:

```typescript
// src/index.ts
import "dotenv/config";
import express from "express";

const app = express();
const PORT = Number(process.env.PORT) || 3000;

app.use(express.json()); // парсим тело запроса как JSON

app.get("/", (req, res) => {
  res.json({ message: "Weather API работает" });
});

app.listen(PORT, () => {
  console.log(`Сервер запущен: http://localhost:${PORT}`);
});
```

`express.json()` — встроенный middleware, который читает тело POST/PUT-запроса и кладёт его в `req.body`. Без него `req.body` будет `undefined`.

## 📖 Типизация Request и Response

Express поставляется с дженерик-типами для маршрутов:

```typescript
import { Request, Response, NextFunction } from "express";

// Полная сигнатура обработчика:
// Request<Params, ResBody, ReqBody, Query>
app.get(
  "/api/weather/:city",
  (
    req: Request<{ city: string }>,
    res: Response
  ) => {
    const { city } = req.params; // TypeScript знает: city — строка
    res.json({ city });
  }
);
```

Параметры дженерика `Request<Params, ResBody, ReqBody, Query>`:
- `Params` — тип `req.params` (параметры маршрута: `:city`)
- `ResBody` — тип тела ответа (что отправляем)
- `ReqBody` — тип `req.body` (тело входящего запроса)
- `Query` — тип `req.query` (строка запроса: `?units=metric`)

На практике указывают только то, что нужно. Остальные параметры оставляют по умолчанию:

```typescript
// Типизированы params и query
app.get(
  "/api/weather/:city",
  (
    req: Request<{ city: string }, unknown, unknown, { units?: string }>,
    res: Response
  ) => {
    const city = req.params.city;
    const units = req.query.units ?? "metric";
    res.json({ city, units });
  }
);
```

## 📖 Первый маршрут: прокси к OpenWeatherMap

Основная идея прокси: клиент отправляет запрос на наш сервер, сервер запрашивает OpenWeatherMap с API-ключом и возвращает результат. Клиент никогда не видит ключ.

```typescript
// src/index.ts
import "dotenv/config";
import express, { Request, Response } from "express";

const app = express();
const PORT = Number(process.env.PORT) || 3000;
const WEATHER_API_KEY = process.env.WEATHER_API_KEY;

if (!WEATHER_API_KEY) {
  console.error("WEATHER_API_KEY не задан");
  process.exit(1);
}

app.use(express.json());

app.get(
  "/api/weather/:city",
  async (req: Request<{ city: string }>, res: Response) => {
    const { city } = req.params;

    const url = `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(city)}&appid=${WEATHER_API_KEY}&units=metric&lang=ru`;

    const response = await fetch(url);
    const data = await response.json();

    if (!response.ok) {
      res.status(response.status).json({ error: data.message });
      return;
    }

    res.json(data);
  }
);

app.listen(PORT, () => {
  console.log(`Сервер запущен: http://localhost:${PORT}`);
});
```

`fetch` доступен в Node.js начиная с версии 18 без установки дополнительных пакетов.

Проверить маршрут можно через curl или браузер:

```bash
curl http://localhost:3000/api/weather/Москва
```

## 📖 Типизация ответа от внешнего API

Сейчас `data` имеет тип `any`. Чтобы TypeScript помогал при работе с ответом, описывают интерфейс:

```typescript
interface WeatherApiResponse {
  name: string;
  main: {
    temp: number;
    feels_like: number;
    humidity: number;
  };
  weather: Array<{
    description: string;
    icon: string;
  }>;
  wind: {
    speed: number;
  };
}

// Тип ответа нашего API — только нужные поля
interface WeatherResponse {
  city: string;
  temp: number;
  feels_like: number;
  description: string;
  humidity: number;
  wind_speed: number;
}
```

```typescript
app.get(
  "/api/weather/:city",
  async (req: Request<{ city: string }>, res: Response) => {
    const { city } = req.params;

    const url = `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(city)}&appid=${WEATHER_API_KEY}&units=metric&lang=ru`;

    const response = await fetch(url);

    if (!response.ok) {
      const error = await response.json() as { message: string };
      res.status(response.status).json({ error: error.message });
      return;
    }

    const data = await response.json() as WeatherApiResponse;

    const result: WeatherResponse = {
      city: data.name,
      temp: data.main.temp,
      feels_like: data.main.feels_like,
      description: data.weather[0].description,
      humidity: data.main.humidity,
      wind_speed: data.wind.speed,
    };

    res.json(result);
  }
);
```

Теперь TypeScript проверяет, что мы обращаемся только к существующим полям ответа.

## 📖 Обработка ошибок в async-обработчиках

Если `fetch` выбросит исключение (нет сети, таймаут), необработанная ошибка уронит сервер. Express не перехватывает ошибки в async-функциях автоматически — нужен явный `try/catch`:

```typescript
app.get(
  "/api/weather/:city",
  async (req: Request<{ city: string }>, res: Response) => {
    try {
      const { city } = req.params;
      // ... запрос к API
      res.json(result);
    } catch (error) {
      console.error("Ошибка при запросе к OpenWeatherMap:", error);
      res.status(500).json({ error: "Внутренняя ошибка сервера" });
    }
  }
);
```

В следующих разделах появится `asyncHandler` — обёртка, которая избавит от повторяющегося `try/catch` в каждом маршруте.

import express, { Request, Response } from "express";

const app = express();

## 🎯 Финальное задание

Доработай прокси: вместо того чтобы возвращать весь ответ OpenWeatherMap, возвращай только нужные поля.

1. Опиши интерфейсы `WeatherApiResponse` и `WeatherResponse`
2. В маршруте приводи `data` к `WeatherApiResponse` через `as`
3. Формируй объект `WeatherResponse` из нужных полей и возвращай его

Ожидаемый ответ:
```json
{
  "city": "Moscow",
  "temp": -3.2,
  "feels_like": -7.1,
  "description": "пасмурно",
  "humidity": 82,
  "wind_speed": 4.5
}
```

Дополнительно: добавь поддержку query-параметра `?units=imperial` — передавай его в запрос к OpenWeatherMap.
