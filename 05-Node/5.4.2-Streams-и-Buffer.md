# 5.4.2 Streams и Buffer

Когда Node.js читает файл через `fs.readFile`, он загружает его целиком в память и только потом отдаёт результат. Для файла в 5 МБ это нормально. Для логов в 2 ГБ или видео — нет: сервер потребляет гигабайты оперативной памяти, а клиент ждёт, пока файл полностью считается.

Streams решают это иначе: данные передаются кусками (chunks), не дожидаясь, пока весь файл окажется в памяти. Файл читается, обрабатывается и отправляется клиенту одновременно — кусок за куском.

## 📖 Buffer

Buffer — это фиксированный блок памяти для хранения бинарных данных. В отличие от строки, Buffer хранит сырые байты без привязки к кодировке.

```typescript
// Создать из строки
const buf = Buffer.from("Привет", "utf-8");
console.log(buf);          // <Buffer d0 9f d1 80 d0 b8 d0 b2 d0 b5 d1 82>
console.log(buf.length);   // 12 (байт, не символов)

// Превратить обратно в строку
console.log(buf.toString("utf-8")); // "Привет"

// Создать пустой буфер заданного размера
const empty = Buffer.alloc(16);    // 16 нулевых байт

// Объединить несколько буферов
const a = Buffer.from("foo");
const b = Buffer.from("bar");
const combined = Buffer.concat([a, b]);
console.log(combined.toString()); // "foobar"
```

Buffer появляется там, где данные ещё не превратились в строку: при чтении файлов, получении данных из сети, работе с криптографией. Streams передают данные именно как Buffer — если не указать кодировку явно.

---

## 📖 Readable Stream

Readable — поток, из которого читают данные. Типичный пример — чтение файла:

```typescript
import { createReadStream } from "fs";

const stream = createReadStream("large-file.csv", { encoding: "utf-8" });

// Вариант 1: через события
stream.on("data", (chunk) => {
  console.log("Получен кусок:", chunk.length, "байт");
});

stream.on("end", () => {
  console.log("Файл прочитан");
});

stream.on("error", (err) => {
  console.error("Ошибка:", err.message);
});
```

```typescript
// Вариант 2: async iteration — чище и безопаснее
async function readFile(path: string): Promise<void> {
  const stream = createReadStream(path, { encoding: "utf-8" });

  for await (const chunk of stream) {
    console.log("Кусок:", chunk);
  }

  console.log("Готово");
}
```

По умолчанию размер куска — 64 КБ. Изменить можно через `highWaterMark`:

```typescript
createReadStream("file.csv", { highWaterMark: 16 * 1024 }) // куски по 16 КБ
```

## 📖 Writable Stream

Writable — поток, в который пишут данные. Типичный пример — запись файла:

```typescript
import { createWriteStream } from "fs";

const stream = createWriteStream("output.txt");

stream.write("Первая строка\n");
stream.write("Вторая строка\n");
stream.end(); // закрываем поток

stream.on("finish", () => {
  console.log("Файл записан");
});
```

В HTTP-контексте `res` в Express — тоже Writable Stream. Можно писать данные в ответ кусками, не формируя весь ответ заранее:

```typescript
import { createReadStream } from "fs";
import { Request, Response } from "express";

export function downloadFile(req: Request, res: Response): void {
  res.setHeader("Content-Type", "text/plain");
  res.setHeader("Content-Disposition", "attachment; filename=data.txt");

  const fileStream = createReadStream("./data.txt");
  fileStream.pipe(res); // читаем файл и пишем прямо в ответ
}
```

## 📖 Transform Stream

Transform — поток, который одновременно читает и пишет, преобразуя данные по ходу. Входные данные поступают с одной стороны, на выходе — уже обработанные.

Стандартный пример — сжатие через `zlib`:

```typescript
import { createReadStream, createWriteStream } from "fs";
import { createGzip } from "zlib";

const input = createReadStream("large-file.csv");
const gzip = createGzip();                         // Transform: сжимает данные
const output = createWriteStream("large-file.csv.gz");

input.pipe(gzip).pipe(output);

output.on("finish", () => {
  console.log("Файл сжат");
});
```

Данные проходят цепочку: `input` → `gzip` (Transform) → `output`. В памяти в любой момент — только один кусок, а не весь файл.

Можно написать свой Transform. Например, конвертер объектов в CSV-строки:

```typescript
import { Transform } from "stream";

interface WeatherRecord {
  city: string;
  temp: number;
  description: string;
}

class ObjectToCsvTransform extends Transform {
  private isFirstChunk = true;

  constructor() {
    super({ objectMode: true }); // принимает объекты, а не Buffer
  }

  _transform(
    record: WeatherRecord,
    _encoding: string,
    callback: () => void
  ): void {
    if (this.isFirstChunk) {
      this.push("city,temp,description\n"); // заголовок
      this.isFirstChunk = false;
    }
    this.push(`${record.city},${record.temp},${record.description}\n`);
    callback();
  }
}
```

`objectMode: true` — поток принимает произвольные объекты вместо Buffer/строк.

## 📖 pipe и pipeline

`pipe` соединяет Readable с Writable и автоматически управляет backpressure — замедляет источник, если получатель не успевает обрабатывать данные:

```typescript
readable.pipe(writable);

// Цепочка через Transform
readable.pipe(transform).pipe(writable);
```

Проблема `pipe` — ошибки нужно перехватывать на каждом потоке отдельно, иначе они теряются. `pipeline` из `node:stream/promises` решает это: пробрасывает ошибку от любого потока и возвращает Promise:

```typescript
import { pipeline } from "node:stream/promises";
import { createReadStream, createWriteStream } from "fs";
import { createGzip } from "zlib";

async function compressFile(src: string, dest: string): Promise<void> {
  await pipeline(
    createReadStream(src),
    createGzip(),
    createWriteStream(dest)
  );
  console.log("Сжатие завершено");
}
```

Если любой из потоков выбросит ошибку — `pipeline` отклонит Promise и закроет все потоки. **Используй `pipeline` вместо `pipe` всегда, когда нужна надёжная обработка ошибок.**

## 📖 Стрим в HTTP-ответе: экспорт CSV

Практический пример для Weather API — маршрут, который отдаёт избранные города как CSV-файл. Вместо того чтобы формировать строку целиком, данные пишутся в ответ построчно:

```typescript
// src/routes/favorites.router.ts
import { Router } from "express";
import { asyncHandler } from "../middleware/asyncHandler";
import { authMiddleware } from "../middleware/auth.middleware";
import { getFavorites } from "../services/favorites.service";
import { PassThrough } from "stream";

const router = Router();

router.get(
  "/export",
  authMiddleware,
  asyncHandler(async (req, res) => {
    const cities = getFavorites();

    res.setHeader("Content-Type", "text/csv; charset=utf-8");
    res.setHeader("Content-Disposition", "attachment; filename=favorites.csv");

    // PassThrough — простой Transform, который не меняет данные;
    // удобен как промежуточный буфер
    const csvStream = new PassThrough();
    csvStream.pipe(res);

    csvStream.write("city\n");
    for (const city of cities) {
      csvStream.write(`${city}\n`);
    }
    csvStream.end();
  })
);

export default router;
```

Для небольших данных разница незаметна, но подход остаётся правильным: `res` — Writable Stream, и писать в него можно напрямую через `pipe`.

## 📖 Когда нужны Streams

Streams добавляют сложности, поэтому стоит понимать, когда они реально нужны, а когда достаточно обычного `await`.

**Используй Streams:**
- Файлы больше нескольких мегабайт (видео, архивы, большие экспорты)
- Данные нужно начать отдавать клиенту до того, как они полностью готовы
- Преобразование данных на лету: сжатие, шифрование, парсинг CSV построчно
- Работа с внешними источниками с неизвестным размером (поток логов, WebSocket)

**Хватает обычного `await`:**
- Небольшие файлы и конфиги (до 1–2 МБ)
- JSON-ответы от API
- Любые данные, которые нужны целиком для обработки (парсинг, поиск по содержимому)

```typescript
// Хватает await — файл маленький, нужен целиком
const config = JSON.parse(await fs.readFile("config.json", "utf-8"));

// Нужен Stream — файл большой, отдаём клиенту по кускам
const fileStream = createReadStream("backup.tar.gz");
fileStream.pipe(res);
```

export async function countLines(filePath: string): Promise<number> {
