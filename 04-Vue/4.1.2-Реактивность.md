# 4.1.2 Реактивность: ref, reactive, computed

Реактивность — главная идея Vue. Ты объявляешь данные как реактивные, и Vue автоматически обновляет шаблон везде, где они используются. Не нужно вручную дёргать DOM — достаточно изменить значение.

---

## 🎯 Что ты узнаешь

- Что такое `ref` и как работает `.value`
- Что такое `reactive` и для каких данных он подходит
- Как работает `computed` и чем отличается от обычной функции
- Почему деструктуризация `reactive` ломает реактивность
- Когда выбирать `ref`, а когда `reactive`

---

## 📖 ref — реактивная переменная

`ref` оборачивает любое значение в реактивный контейнер. При изменении `.value` Vue перерисовывает шаблон.

```vue
<script setup lang="ts">
import { ref } from "vue";

const temperature = ref(20); // ref<number>
const city = ref("Москва");  // ref<string>
const isLoading = ref(false); // ref<boolean>

function increment() {
  temperature.value++; // изменяем через .value
}
</script>

<template>
  <p>{{ temperature }}°C</p>  <!-- в шаблоне .value не нужен -->
  <button @click="increment">+1°C</button>
</template>
```

**Почему `.value`?** JavaScript не умеет отслеживать изменения примитивов напрямую. `ref` оборачивает значение в объект — и Vue наблюдает за этим объектом. В шаблоне Vue автоматически «разворачивает» ref, поэтому там `.value` писать не нужно.

### ref с TypeScript

Тип выводится автоматически, но можно указать явно:

```typescript
import { ref } from "vue";
import type { WeatherData } from "@/types/weather";

// Тип выводится из начального значения
const temperature = ref(20); // Ref<number>

// Явное указание типа — нужно когда начальное значение null
const weatherData = ref<WeatherData | null>(null);

// После загрузки присваиваем данные
weatherData.value = {
  city: "Москва",
  temperature: 20,
  // ...
};
```

## 📖 reactive — реактивный объект

`reactive` делает реактивным целый объект. Не нужно писать `.value` — обращаешься к полям напрямую:

```vue
<script setup lang="ts">
import { reactive } from "vue";

const weather = reactive({
  city: "Москва",
  temperature: 20,
  humidity: 65,
  isLoading: false,
});

function setCity(name: string) {
  weather.city = name; // без .value — просто присваиваем
  weather.isLoading = true;
}
</script>

<template>
  <p>{{ weather.city }}: {{ weather.temperature }}°C</p>
</template>
```

### ⚠️ Деструктуризация ломает реактивность

Самая распространённая ошибка с `reactive`:

```typescript
const weather = reactive({ temperature: 20, city: "Москва" });

// ❌ Деструктуризация — реактивность теряется
const { temperature, city } = weather;
temperature; // просто число, не реактивное

// ✅ Используй toRefs если нужна деструктуризация
import { reactive, toRefs } from "vue";
const { temperature, city } = toRefs(weather);
temperature.value; // теперь реактивно
```

Почему так происходит: при деструктуризации ты копируешь примитивное значение — связь с объектом `weather` теряется, Vue больше не видит изменений.

## 📖 computed — вычисляемые значения

`computed` создаёт значение, которое автоматически пересчитывается, когда меняются его зависимости:

```vue
<script setup lang="ts">
import { ref, computed } from "vue";

const temperature = ref(20);
const unit = ref<"celsius" | "fahrenheit">("celsius");

// Пересчитывается автоматически при изменении temperature или unit
const displayTemp = computed(() => {
  if (unit.value === "fahrenheit") {
    return `${Math.round(temperature.value * 9/5 + 32)}°F`;
  }
  return `${temperature.value}°C`;
});
</script>

<template>
  <p>{{ displayTemp }}</p>  <!-- обновится сам при смене unit или temperature -->
</template>
```

### computed vs функция в шаблоне

```vue
<script setup lang="ts">
import { ref, computed } from "vue";

const temperature = ref(20);

// Функция — вызывается при КАЖДОМ рендере
function getDisplay() {
  console.log("вызов функции");
  return `${temperature.value}°C`;
}

// computed — пересчитывается только когда temperature изменится
const display = computed(() => {
  console.log("вычисление computed");
  return `${temperature.value}°C`;
});
</script>
```

Если компонент перерисовывается по другой причине (изменился несвязанный `ref`), функция вызовется снова, а `computed` вернёт кэшированное значение — пересчёт произойдёт только при изменении `temperature`.

### computed с сеттером

`computed` можно сделать записываемым — через объект с `get` и `set`:

```typescript
const temperatureF = computed({
  get() {
    return Math.round(temperature.value * 9/5 + 32);
  },
  set(fahrenheit: number) {
    temperature.value = Math.round((fahrenheit - 32) * 5/9);
  },
});

// Теперь можно читать и писать:
temperatureF.value;      // читаем — получаем °F
temperatureF.value = 68; // пишем — temperature.value автоматически станет 20
```

## 📖 ref vs reactive — когда что выбирать

На практике чаще используют `ref` — он работает с любыми типами и поведение предсказуемо:

| | `ref` | `reactive` |
|---|---|---|
| Примитивы | ✅ | ❌ |
| Объекты | ✅ | ✅ |
| Доступ к значению | `x.value` | `x.field` |
| Деструктуризация | ✅ безопасна | ⚠️ ломает реактивность |
| Замена всего объекта | ✅ `x.value = {}` | ❌ теряет реактивность |

```typescript
// Предпочтительный подход — ref для всего
const city = ref("Москва");
const temperature = ref(20);
const weatherData = ref<WeatherData | null>(null);

// reactive удобен когда данные логически связаны и ты не будешь их деструктурировать
const form = reactive({
  city: "",
  unit: "metric" as "metric" | "imperial",
});
```

## 🔧 Практика 4.1.2.1: ref в WeatherCard

Добавим реактивность в `WeatherCard.vue` — переключение единиц температуры между °C и °F.

**Шаг 1: Обнови `src/components/WeatherCard.vue`**

```vue
<script setup lang="ts">
import { ref, computed } from "vue";
import type { WeatherData } from "../types/weather";

const weatherData = ref<WeatherData>({
  city: "Москва",
  country: "RU",
  temperature: 20,
  feelsLike: 18,
  humidity: 65,
  description: "Облачно",
  icon: "04d",
  timestamp: Date.now(),
});

const unit = ref<"celsius" | "fahrenheit">("celsius");

const displayTemp = computed(() => {
  if (unit.value === "fahrenheit") {
    return `${Math.round(weatherData.value.temperature * 9/5 + 32)}°F`;
  }
  return `${weatherData.value.temperature}°C`;
});

const displayFeelsLike = computed(() => {
  if (unit.value === "fahrenheit") {
    return `${Math.round(weatherData.value.feelsLike * 9/5 + 32)}°F`;
  }
  return `${weatherData.value.feelsLike}°C`;
});

function toggleUnit() {
  unit.value = unit.value === "celsius" ? "fahrenheit" : "celsius";
}
</script>

<template>
  <div class="weather-card">
    <div class="weather-card__header">
      <div>
        <h2 class="weather-card__city">{{ weatherData.city }}</h2>
        <span class="weather-card__country">{{ weatherData.country }}</span>
      </div>
      <button class="weather-card__unit-btn" @click="toggleUnit">
        {{ unit === "celsius" ? "°C" : "°F" }}
      </button>
    </div>

    <div class="weather-card__body">
      <p class="weather-card__temp">{{ displayTemp }}</p>
      <p class="weather-card__description">{{ weatherData.description }}</p>
    </div>

    <div class="weather-card__footer">
      <span>💧 {{ weatherData.humidity }}%</span>
      <span>🌡 Ощущается как {{ displayFeelsLike }}</span>
    </div>
  </div>
</template>
```

## 🔧 Практика 4.1.2.2: reactive для состояния загрузки

Добавим в `App.vue` реактивное состояние с `reactive` — будем имитировать загрузку данных.

**Шаг 1: Обнови `src/App.vue`**

```vue
<script setup lang="ts">
import { reactive } from "vue";
import WeatherCard from "./components/WeatherCard.vue";
import LoadingSpinner from "./components/LoadingSpinner.vue";

const appState = reactive({
  isLoading: true,
  error: null as string | null,
  city: "Москва",
});

// Имитируем загрузку — через 1.5 секунды скрываем спиннер
setTimeout(() => {
  appState.isLoading = false;
}, 1500);
</script>

<template>
  <main>
    <h1>🌤 Погода</h1>

    <LoadingSpinner v-if="appState.isLoading" />
    <WeatherCard v-else />

    <p v-if="appState.error" class="error">{{ appState.error }}</p>
  </main>
</template>

<style scoped>
main {
  max-width: 480px;
  margin: 40px auto;
  padding: 0 16px;
}

h1 {
  font-size: 20px;
  font-weight: 600;
  margin-bottom: 20px;
  color: #374151;
}

.error {
  color: #ef4444;
  font-size: 14px;
  margin-top: 12px;
}
</style>
```

**Шаг 2: Посмотри на `v-if` / `v-else`**

`v-if="appState.isLoading"` — директива условного рендеринга. Пока `isLoading === true`, показывается спиннер. Как только `setTimeout` меняет `isLoading` на `false` — Vue убирает спиннер и показывает карточку. Подробно про директивы — в разделе 4.1.3.

## 📝 Итоговое задание 4.1.2

**Задача:** Добавить в `WeatherCard.vue` реактивный список избранных городов.

### Требования:

1. Добавь `ref` для хранения массива избранных городов:

   ```typescript
   const favorites = ref<string[]>([]);
   ```

2. Добавь `computed` для проверки, находится ли текущий город в избранном:

   ```typescript
   const isFavorite = computed(() =>
     favorites.value.includes(weatherData.value.city)
   );
   ```

3. Добавь функцию переключения избранного:

   ```typescript
   function toggleFavorite() {
     const city = weatherData.value.city;
     if (isFavorite.value) {
       favorites.value = favorites.value.filter((c) => c !== city);
     } else {
       favorites.value.push(city);
     }
   }
   ```

4. Добавь в шаблон кнопку с иконкой звезды — она должна меняться в зависимости от `isFavorite`:

   ```vue
   <button @click="toggleFavorite" class="weather-card__fav-btn">
     {{ isFavorite ? "★" : "☆" }}
   </button>
   ```

5. Добавь `computed` для счётчика избранных и выведи его под карточкой:

   ```typescript
   const favoritesCount = computed(() => favorites.value.length);
   ```

### Проверка:

- Нажатие на звезду добавляет город в избранное (★)
- Повторное нажатие убирает (☆)
- Счётчик обновляется автоматически
