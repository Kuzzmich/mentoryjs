# 5.2.6 Загрузка файлов (Multer)

Загрузка файлов отличается от обычных JSON-запросов: данные передаются в формате `multipart/form-data`, и стандартный `express.json()` их не читает. Multer — middleware для Express, который разбирает такие запросы, сохраняет файлы и кладёт информацию о них в `req.file`.

---

## 🎯 Что ты узнаешь

- Как настроить Multer для сохранения файлов на диск
- Как валидировать тип и размер загружаемого файла
- Как раздавать загруженные файлы по публичной ссылке
- Как сделать маршрут `POST /api/users/avatar`

---

## 📖 Установка и базовая настройка

```bash
npm install multer
npm install -D @types/multer
```

Multer настраивается через `storage` — объект, который определяет, куда и под каким именем сохранять файл:

```typescript
import multer from "multer";
import path from "path";
import crypto from "crypto";

const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    cb(null, "uploads/"); // папка для сохранения
  },
  filename: (req, file, cb) => {
    // Генерируем уникальное имя: случайный hex + оригинальное расширение
    const ext = path.extname(file.originalname);
    const name = crypto.randomBytes(16).toString("hex");
    cb(null, `${name}${ext}`);
  },
});

export const upload = multer({ storage });
```

Папку `uploads/` нужно создать заранее или создавать при старте приложения.

---

## 📖 Типы загрузки: single, array, fields

```typescript
// Один файл с именем поля "avatar"
upload.single("avatar")

// До 5 файлов с именем поля "photos"
upload.array("photos", 5)

// Разные поля
upload.fields([
  { name: "avatar", maxCount: 1 },
  { name: "documents", maxCount: 3 },
])
```

После `upload.single("avatar")` файл доступен в `req.file`, после `upload.array` — в `req.files`.

## 📖 Валидация типа и размера

Multer принимает `fileFilter` — функцию, которая разрешает или отклоняет файл, и `limits` для ограничений:

```typescript
import multer, { FileFilterCallback } from "multer";

const ALLOWED_MIME_TYPES = ["image/jpeg", "image/png", "image/webp"];
const MAX_FILE_SIZE = 5 * 1024 * 1024; // 5 MB

const fileFilter = (
  req: Express.Request,
  file: Express.Multer.File,
  cb: FileFilterCallback
) => {
  if (ALLOWED_MIME_TYPES.includes(file.mimetype)) {
    cb(null, true); // принять файл
  } else {
    cb(new Error("Допустимые форматы: JPEG, PNG, WebP"));
  }
};

export const upload = multer({
  storage,
  fileFilter,
  limits: {
    fileSize: MAX_FILE_SIZE,
  },
});
```

Если файл превышает `fileSize`, Multer выбрасывает `MulterError` с кодом `LIMIT_FILE_SIZE`. Её нужно обрабатывать в error middleware:

```typescript
// src/middleware/errorHandler.ts
import multer from "multer";

export function errorHandler(err: Error, req: Request, res: Response, next: NextFunction): void {
  if (err instanceof multer.MulterError) {
    if (err.code === "LIMIT_FILE_SIZE") {
      res.status(400).json({ error: "Файл слишком большой. Максимум 5 MB" });
      return;
    }
    res.status(400).json({ error: err.message });
    return;
  }

  if (err instanceof AppError) {
    res.status(err.statusCode).json({ error: err.message });
    return;
  }

  res.status(500).json({ error: "Внутренняя ошибка сервера" });
}
```

## 📖 Публичный доступ к файлам

Загруженные файлы хранятся на диске. Чтобы клиент мог их получить по URL, нужно раздавать папку `uploads/` как статику:

```typescript
// src/index.ts
app.use("/uploads", express.static("uploads"));
// GET /uploads/abc123.jpg → файл из папки uploads/
```

Тогда ссылка на аватар будет: `http://localhost:3000/uploads/abc123.jpg`

## 📖 Маршрут загрузки аватара

```typescript
// src/routes/users.router.ts
import { Router } from "express";
import { upload } from "../middleware/upload";
import { authMiddleware } from "../middleware/auth.middleware";
import { asyncHandler } from "../middleware/asyncHandler";
import { uploadAvatar } from "../controllers/users.controller";

const router = Router();

router.post(
  "/avatar",
  authMiddleware,
  upload.single("avatar"),
  asyncHandler(uploadAvatar)
);

export default router;
```

```typescript
// src/controllers/users.controller.ts
import { Request, Response } from "express";
import { AppError } from "../middleware/AppError";

export async function uploadAvatar(req: Request, res: Response): Promise<void> {
  if (!req.file) {
    throw new AppError("Файл не загружен", 400);
  }

  const avatarUrl = `/uploads/${req.file.filename}`;

  // В реальном проекте здесь обновляем запись пользователя в БД:
  // await updateUserAvatar(req.user!.id, avatarUrl);

  res.json({
    message: "Аватар загружен",
    url: avatarUrl,
  });
}
```

Подключи роутер в `index.ts`:

```typescript
app.use("/uploads", express.static("uploads"));

```

Проверить через curl:

```bash
curl -X POST http://localhost:3000/api/users/avatar \
  -H "Authorization: Bearer <token>" \
  -F "avatar=@/path/to/photo.jpg"
```

## 📖 Хранение в памяти (альтернатива)

Если не нужно сохранять файл на диск — например, сразу передаёшь его в облако — используй `memoryStorage`:

```typescript
const upload = multer({ storage: multer.memoryStorage() });
```

Тогда файл доступен как `Buffer` через `req.file.buffer`. Это удобно при загрузке в S3 или другое облачное хранилище.
