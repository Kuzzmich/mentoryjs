# 4.2.2 Слоты и динамические компоненты

Props передают данные. Слоты передают разметку. Это позволяет создавать компоненты-обёртки, которые управляют структурой и поведением, а содержимое внутрь вставляет родитель.

---

## 🎯 Что ты узнаешь

- Слот по умолчанию — как `children` в React
- Именованные слоты — несколько точек вставки в одном компоненте
- Scoped slots — когда дочерний компонент передаёт данные в слот
- `<component :is="...">` — динамический выбор компонента

---

## 📖 Слот по умолчанию

Слот — это место в шаблоне компонента, куда родитель вставляет свою разметку:

```vue
<!-- src/components/Card.vue -->
<template>
  <div class="card">
    <slot />  <!-- сюда попадёт содержимое от родителя -->
  </div>
</template>
```

```vue
<!-- App.vue -->
<Card>
  <p>Любое содержимое</p>
  <WeatherInfo />
</Card>
```

### Fallback-контент

Если родитель ничего не передал — слот показывает содержимое по умолчанию:

```vue
<template>
  <div class="card">
    <slot>
      <p class="card__empty">Нет данных</p>  <!-- покажется если слот пуст -->
    </slot>
  </div>
</template>
```

---

## 📖 Именованные слоты

Когда нужно несколько точек вставки — используй именованные слоты:

```vue
<!-- src/components/BaseCard.vue -->
<template>
  <div class="base-card">
    <header class="base-card__header">
      <slot name="header" />
    </header>

    <div class="base-card__body">
      <slot />  <!-- слот без имени — default -->
    </div>

    <footer class="base-card__footer">
      <slot name="footer" />
    </footer>
  </div>
</template>
```

Заполнение именованных слотов через `v-slot` (сокращение `#`):

```vue
<!-- App.vue -->
<BaseCard>
  <template #header>
    <h2>Москва</h2>
    <span>RU</span>
  </template>

  <!-- default слот — без template -->
  <p>20°C, облачно</p>

  <template #footer>
    <span>Обновлено: 14:30</span>
    <button>Обновить</button>
  </template>
</BaseCard>
```

---

## 📖 Scoped slots

Обычно данные идут от родителя к дочернему компоненту через props. Scoped slot переворачивает это: дочерний компонент передаёт данные в слот, а родитель решает как их отрендерить.

```vue
<!-- src/components/WeatherList.vue -->
<script setup lang="ts">
import type { WeatherData } from "../types/weather";

defineProps<{ items: WeatherData[] }>();
</script>

<template>
  <ul class="weather-list">
    <li v-for="item in items" :key="item.city">
      <!-- передаём item в слот как scoped переменную -->
      <slot :item="item" />
    </li>
  </ul>
</template>
```

```vue
<!-- App.vue — родитель получает item из слота -->
<WeatherList :items="forecasts">
  <template #default="{ item }">
    <strong>{{ item.city }}</strong>
    <span>{{ item.temperature }}°C</span>
  </template>
</WeatherList>
```

Scoped slots нужны когда компонент управляет данными и их обходом, а родитель — внешним видом каждого элемента. Типичные примеры: таблицы, списки с кастомным рендером, дропдауны.

---

## 📖 Динамические компоненты: `<component :is>`

`<component :is="...">` рендерит разный компонент в зависимости от значения переменной:

```vue
<script setup lang="ts">
import WeatherCard from "./components/WeatherCard.vue";
import LoadingSpinner from "./components/LoadingSpinner.vue";
import ErrorMessage from "./components/ErrorMessage.vue";

// Тип — компонент Vue или строка HTML-тега
const currentView = computed(() => {
  if (isLoading.value) return LoadingSpinner;
  if (error.value) return ErrorMessage;
  return WeatherCard;
});
</script>

<template>
  <component :is="currentView" v-bind="currentProps" />
</template>
```

Это чище чем цепочка `v-if / v-else-if / v-else` когда компонентов несколько.

### Сохранение состояния: `<KeepAlive>`

По умолчанию при смене компонента старый уничтожается. `<KeepAlive>` кэширует его:

```vue
<template>
  <!-- При переключении вкладок состояние WeatherCard сохраняется -->
  <KeepAlive>
    <component :is="activeTab" />
  </KeepAlive>
</template>
```
