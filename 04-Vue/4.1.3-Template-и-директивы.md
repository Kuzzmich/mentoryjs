# 4.1.3 Template и директивы

Шаблон Vue — это HTML с суперспособностями. Ты пишешь привычную разметку, добавляешь директивы — и Vue сам обновляет DOM при изменении данных.

---

## 🎯 Что ты узнаешь

- Как работает интерполяция `{{ }}` и привязка атрибутов `:bind`
- `v-if` / `v-else` / `v-show` — когда что использовать
- `v-for` — рендеринг списков и зачем нужен `:key`
- `v-model` — двустороннее связывание с полями ввода
- `v-on` / `@` — обработка событий и модификаторы

---

## 📖 Интерполяция и привязка атрибутов

### Интерполяция `{{ }}`

Выводит значение переменной в текст. Внутри можно использовать любое JavaScript-выражение:

```vue
<script setup lang="ts">
import { ref } from "vue";

const city = ref("Москва");
const temperature = ref(20);
const isOnline = ref(true);
</script>

<template>
  <p>{{ city }}</p>                          <!-- Москва -->
  <p>{{ temperature }}°C</p>                 <!-- 20°C -->
  <p>{{ isOnline ? "онлайн" : "офлайн" }}</p>  <!-- онлайн -->
  <p>{{ city.toUpperCase() }}</p>            <!-- МОСКВА -->
</template>
```

Интерполяция работает только в текстовом содержимом тегов. Для атрибутов — нужен `v-bind`.

### Привязка атрибутов: `v-bind` и сокращение `:`

`v-bind` передаёт значение переменной в атрибут HTML-тега. Сокращение `:` используется везде:

```vue
<script setup lang="ts">
const iconCode = ref("04d");
const isDisabled = ref(false);
const cardClass = ref("weather-card--sunny");
</script>

<template>
  <!-- Полная запись -->
  <img v-bind:src="`https://openweathermap.org/img/wn/${iconCode}@2x.png`" />

  <!-- Сокращение : — используй всегда -->
  <img :src="`https://openweathermap.org/img/wn/${iconCode}@2x.png`" :alt="iconCode" />

  <!-- Динамический класс -->
  <div :class="cardClass">...</div>

  <!-- Отключение кнопки -->
  <button :disabled="isDisabled">Обновить</button>
</template>
```

### Динамические классы и стили

Для классов и стилей Vue поддерживает объектный и массивный синтаксис:

```vue
<script setup lang="ts">
const temperature = ref(20);
const isLoading = ref(false);
</script>

<template>
  <!-- Объект: класс добавляется если условие true -->
  <div :class="{
    'weather-card': true,
    'weather-card--loading': isLoading,
    'weather-card--hot': temperature > 30,
    'weather-card--cold': temperature < 0,
  }">...</div>

  <!-- Инлайн-стиль через объект -->
  <p :style="{ color: temperature > 30 ? '#ef4444' : '#111827' }">
    {{ temperature }}°C
  </p>
</template>
```

## 📖 Условный рендеринг: v-if, v-else, v-show

### v-if и v-else

`v-if` добавляет или удаляет элемент из DOM:

```vue
<template>
  <LoadingSpinner v-if="isLoading" />
  <WeatherCard v-else-if="weatherData" />
  <p v-else class="error">Данные недоступны</p>
</template>
```

Элемент с `v-if="false"` полностью отсутствует в DOM — не занимает память и не влияет на производительность.

### v-show

`v-show` не удаляет элемент — только скрывает через `display: none`:

```vue
<template>
  <!-- Элемент есть в DOM, но скрыт через CSS -->
  <div v-show="isExpanded" class="details">
    <p>Влажность: {{ humidity }}%</p>
    <p>Давление: {{ pressure }} гПа</p>
  </div>
</template>
```

### Когда что использовать

| | `v-if` | `v-show` |
|---|---|---|
| Что делает | Удаляет/создаёт DOM-элемент | Скрывает через `display: none` |
| Когда использовать | Редкие переключения | Частые переключения |
| Первый рендер | Ленивый (не рендерит при `false`) | Всегда рендерит |
| Пример | Экран загрузки / состояние ошибки | Выпадающее меню, детали карточки |

## 📖 Рендеринг списков: v-for

`v-for` рендерит элемент для каждого элемента массива:

```vue
<script setup lang="ts">
import { ref } from "vue";

const cities = ref(["Москва", "Санкт-Петербург", "Казань"]);
</script>

<template>
  <!-- `:key` обязателен — помогает Vue отслеживать элементы -->
  <ul>
    <li v-for="city in cities" :key="city">
      {{ city }}
    </li>
  </ul>
</template>
```

С индексом и объектами:

```vue
<script setup lang="ts">
import type { WeatherData } from "../types/weather";

const forecasts = ref<WeatherData[]>([...]);
</script>

<template>
  <!-- Индекс как второй аргумент -->
  <div v-for="(forecast, index) in forecasts" :key="forecast.city">
    <span>{{ index + 1 }}. {{ forecast.city }}</span>
    <span>{{ forecast.temperature }}°C</span>
  </div>
</template>
```

### Зачем нужен `:key`

Vue использует `:key` для отслеживания элементов при обновлении списка. Без него Vue может неправильно переиспользовать DOM-элементы — особенно при удалении или перестановке элементов.

```vue
<!-- ❌ Плохо — Vue не может отслеживать элементы -->
<li v-for="city in cities">{{ city }}</li>

<!-- ❌ Плохо — индекс нестабилен при удалении -->
<li v-for="(city, i) in cities" :key="i">{{ city }}</li>

<!-- ✅ Хорошо — уникальный стабильный идентификатор -->
<li v-for="city in cities" :key="city">{{ city }}</li>
<li v-for="forecast in forecasts" :key="forecast.city">...</li>
```

## 📖 Двустороннее связывание: v-model

`v-model` синхронизирует значение реактивной переменной с полем ввода в обе стороны — пользователь меняет поле, переменная обновляется; переменная меняется, поле обновляется:

```vue
<script setup lang="ts">
import { ref } from "vue";

const searchCity = ref("");
const selectedUnit = ref<"celsius" | "fahrenheit">("celsius");
</script>

<template>
  <!-- Текстовое поле -->
  <input v-model="searchCity" type="text" placeholder="Введи город" />
  <p>Ищем: {{ searchCity }}</p>

  <!-- Выпадающий список -->
  <select v-model="selectedUnit">
    <option value="celsius">°C</option>
    <option value="fahrenheit">°F</option>
  </select>
</template>
```

`v-model` — это сокращение для `:value` + `@input`:

```vue
<!-- Эти записи эквивалентны -->
<input v-model="searchCity" />
<input :value="searchCity" @input="searchCity = $event.target.value" />
```

### Модификаторы v-model

```vue
<!-- .trim — убирает пробелы по краям -->
<input v-model.trim="searchCity" />

<!-- .number — преобразует в число -->
<input v-model.number="temperature" type="number" />

<!-- .lazy — обновляет только при потере фокуса (change вместо input) -->
<input v-model.lazy="searchCity" />
```

## 📖 Обработка событий: v-on и @

`v-on` подписывается на DOM-события. Сокращение `@` используется всегда:

```vue
<script setup lang="ts">
import { ref } from "vue";

const count = ref(0);

function handleSearch(city: string) {
  console.log("Ищем:", city);
}
</script>

<template>
  <!-- Инлайн-выражение -->
  <button @click="count++">Нажато: {{ count }}</button>

  <!-- Вызов функции -->
  <button @click="handleSearch('Москва')">Найти</button>

  <!-- Функция без скобок получает event автоматически -->
  <input @keydown="(e) => console.log(e.key)" />
</template>
```

### Модификаторы событий

Модификаторы заменяют типичные `event.preventDefault()` и `event.stopPropagation()`:

```vue
<template>
  <!-- Отменить стандартное поведение формы -->
  <form @submit.prevent="handleSubmit">...</form>

  <!-- Остановить всплытие события -->
  <button @click.stop="handleClick">Кнопка</button>

  <!-- Сработает только при нажатии Enter -->
  <input @keyup.enter="handleSearch" />

  <!-- Сработает только один раз -->
  <button @click.once="loadData">Загрузить</button>
</template>
```

## ✅ Чеклист раздела

После изучения темы ты должен уметь:

- [ ] Выводить данные через `{{ }}` и использовать выражения внутри
- [ ] Привязывать атрибуты через `:bind` (`:src`, `:class`, `:disabled`)
- [ ] Использовать объектный синтаксис `:class` для динамических классов
- [ ] Применять `v-if` / `v-else-if` / `v-else` для условного рендеринга
- [ ] Объяснить разницу между `v-if` и `v-show` и выбирать нужное
- [ ] Рендерить списки через `v-for` с обязательным `:key`
- [ ] Связывать поля ввода с данными через `v-model`
- [ ] Обрабатывать события через `@click`, `@keyup.enter`
- [ ] Использовать модификаторы событий: `.prevent`, `.stop`, `.once`
