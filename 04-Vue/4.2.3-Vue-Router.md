# 4.2.3 Vue Router

Vue Router — официальная библиотека маршрутизации для Vue. Вместо перезагрузки страниц браузер переключается между представлениями, URL обновляется, кнопки «назад/вперёд» работают — всё это без обращения к серверу.

---

## 🎯 Что ты узнаешь

- Установка и базовая настройка роутера
- `<RouterView>` и `<RouterLink>` — основные компоненты
- Динамические маршруты: `/weather/:city`
- `useRoute` и `useRouter` — навигация из компонента
- Навигационные гарды — защита маршрутов

---

## 📖 Установка и настройка

Если при создании проекта через `npm create vue@latest` роутер не был выбран, добавь его:

```bash
npm install vue-router@4
```

**Создай `src/router/index.ts`:**

```typescript
import { createRouter, createWebHistory } from "vue-router";
import HomeView from "../views/HomeView.vue";
import WeatherView from "../views/WeatherView.vue";

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: "/",
      name: "home",
      component: HomeView,
    },
    {
      path: "/weather/:city",  // :city — динамический параметр
      name: "weather",
      component: WeatherView,
    },
    {
      path: "/:pathMatch(.*)*",  // 404 — любой неизвестный путь
      name: "not-found",
      component: () => import("../views/NotFoundView.vue"),  // ленивая загрузка
    },
  ],
});

export default router;
```

### createWebHistory vs createWebHashHistory

`createWebHistory` использует HTML5 History API — URL выглядят как `/weather/Москва`. Для этого нужна настройка сервера: nginx или Apache должны отдавать `index.html` для любого пути, иначе при обновлении страницы на `/weather/Москва` сервер вернёт 404. На локальном Vite-сервере это работает автоматически.

`createWebHashHistory` использует хэш: `/#/weather/Москва`. Не требует настройки сервера — можно деплоить как статику куда угодно. Но выглядит некрасиво и хуже для SEO.

### CSR и SEO

Приложение на Vue Router работает в режиме **CSR (Client-Side Rendering)** — сервер отдаёт пустой HTML, весь контент строит JavaScript в браузере. Поисковые боты видят пустую страницу и не могут проиндексировать содержимое.

При **SSR (Server-Side Rendering)** сервер сам рендерит HTML с уже готовым контентом — поисковик индексирует страницу как обычно.

| | CSR | SSR |
|---|---|---|
| Первый HTML | Пустой `<div>` | Готовый контент |
| SEO | ❌ Поисковик видит пустую страницу | ✅ Контент индексируется |
| Скорость первого экрана | Медленнее — ждём JS | Быстрее — HTML уже есть |
| Сложность | Проще | Нужен Node-сервер |

Для погодного приложения SEO не нужен — пользователь сам открывает его, не находит через поиск. CSR подходит.

Если понадобится SEO — смотри в сторону **Nuxt.js**: это фреймворк над Vue с SSR из коробки, который использует тот же Vue Router и Pinia, что ты изучаешь сейчас.

**Подключи в `src/main.ts`:**

```typescript
import { createApp } from "vue";
import App from "./App.vue";
import router from "./router";

createApp(App).use(router).mount("#app");
```

## 📖 RouterView и RouterLink

`<RouterView>` — место в шаблоне, куда роутер вставляет текущий компонент страницы:

```vue
<!-- App.vue -->
<template>
  <nav>
    <RouterLink to="/">Главная</RouterLink>
    <RouterLink to="/weather/Москва">Москва</RouterLink>
  </nav>

  <RouterView />  <!-- здесь рендерится HomeView или WeatherView -->
</template>
```

`<RouterLink>` рендерит `<a>` но перехватывает клик и меняет URL без перезагрузки. Автоматически добавляет класс `router-link-active` на активную ссылку.

### Именованные маршруты

Лучше ссылаться на маршрут по имени, а не по пути — при изменении URL обновлять придётся только роутер:

```vue
<RouterLink :to="{ name: 'weather', params: { city: 'Москва' } }">
  Москва
</RouterLink>
```

## 📖 Динамические маршруты и useRoute

`useRoute` даёт доступ к текущему маршруту — параметрам, query, имени:

```vue
<!-- src/views/WeatherView.vue -->
<script setup lang="ts">
import { computed } from "vue";
import { useRoute } from "vue-router";
import { useWeather } from "../composables/useWeather";

const route = useRoute();

// route.params.city — строка из URL /weather/:city
const city = computed(() => route.params.city as string);

const { weatherData, isLoading, error } = useWeather(city);
</script>

<template>
  <h1>Погода: {{ city }}</h1>
  <WeatherCard v-if="weatherData" :city="city" :unit="unit" />
</template>
```

### Query-параметры

```typescript
// URL: /weather/Москва?unit=fahrenheit
const unit = computed(() => route.query.unit as string || "celsius");
```

## 📖 Навигация из компонента: useRouter

`useRouter` позволяет программно переходить между страницами:

```vue
<script setup lang="ts">
import { useRouter } from "vue-router";

const router = useRouter();

function goToCity(city: string) {
  // Переход по имени маршрута
  router.push({ name: "weather", params: { city } });
}

function goBack() {
  router.back();
}

// Заменить текущую запись в истории (кнопка «назад» не вернёт)
function redirectHome() {
  router.replace({ name: "home" });
}
</script>
```

### `push` vs `replace`

- `router.push()` — добавляет в историю браузера, кнопка «назад» работает
- `router.replace()` — заменяет текущую запись, «назад» ведёт на страницу до неё

## 📖 Навигационные гарды

Гарды — это перехватчики, которые срабатывают до перехода и могут его отменить или перенаправить.

### Глобальный гард

```typescript
// src/router/index.ts
router.beforeEach((to, from) => {
  // Запретить переход если город не указан
  if (to.name === "weather" && !to.params.city) {
    return { name: "home" };  // перенаправить на главную
  }
  // return true или undefined — разрешить переход
});
```

### Гард на уровне маршрута

```typescript
const routes = [
  {
    path: "/weather/:city",
    component: WeatherView,
    beforeEnter: (to) => {
      // Валидация параметра — только кириллица и латиница
      const city = to.params.city as string;
      if (city.length < 2) {
        return { name: "home" };
      }
    },
  },
];
```

### Гард внутри компонента

```vue
<script setup lang="ts">
import { onBeforeRouteLeave } from "vue-router";

onBeforeRouteLeave((to, from) => {
  const confirmed = window.confirm("Уйти со страницы?");
  if (!confirmed) return false;  // отменить переход
});
</script>
```
