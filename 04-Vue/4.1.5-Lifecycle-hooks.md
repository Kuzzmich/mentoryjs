# 4.1.5 Lifecycle hooks

Каждый компонент Vue проходит несколько этапов: создание, монтирование в DOM (`onMounted`), обновления при изменении данных (`onUpdated`) и уничтожение (`onUnmounted`). Lifecycle hooks позволяют запускать код на нужном этапе — например, загрузить данные при появлении компонента или остановить таймер при его удалении.

---

## 🎯 Что ты узнаешь

- Какие этапы проходит компонент Vue
- `onMounted` — загрузка данных и работа с DOM
- `onUnmounted` — очистка ресурсов
- `onUpdated` — реакция после перерисовки
- Когда lifecycle hooks, а когда `watch` или `computed`

---

## 📖 Жизненный цикл компонента

```
setup()          ← здесь объявляются ref, computed, функции
    │
onBeforeMount()  ← компонент ещё не в DOM
    │
onMounted()      ← компонент появился в DOM ✅ загрузка данных, таймеры
    │
onBeforeUpdate() ← данные изменились, перерисовка ещё не началась
    │
onUpdated()      ← DOM обновлён ✅ нужно редко, обычно хватает computed
    │
onBeforeUnmount()← компонент ещё в DOM, но уже будет удалён
    │
onUnmounted()    ← компонент удалён из DOM ✅ очистка таймеров, подписок
```

В `<script setup>` сам код внутри тега — это и есть `setup()`. Хуки вызываются внутри него.

---

## 📖 onMounted

Срабатывает когда компонент добавлен в DOM. Основное применение — загрузка данных и прямая работа с DOM-элементами:

```vue
<script setup lang="ts">
import { ref, onMounted } from "vue";

const weather = ref<WeatherData | null>(null);
const isLoading = ref(false);
const error = ref<string | null>(null);

onMounted(async () => {
  isLoading.value = true;
  try {
    const response = await fetch(
      `https://api.openweathermap.org/data/2.5/weather?q=Москва&appid=KEY`
    );
    weather.value = await response.json();
  } catch (e) {
    error.value = "Не удалось загрузить данные";
  } finally {
    isLoading.value = false;
  }
});
</script>
```

### Почему не в setup напрямую?

```vue
<script setup lang="ts">
// ❌ Проблема: запрос начинается до монтирования,
// но главное — при SSR (серверном рендеринге) нет DOM и нет смысла
// делать запросы до того, как компонент нужен пользователю
const data = await fetch("/api/weather"); // работает, но подход не оптимальный

// ✅ Правильно: явно говорим "загрузи когда компонент появился"
onMounted(async () => {
  const data = await fetch("/api/weather");
});
</script>
```

## 📖 onUnmounted

Срабатывает когда компонент удалён из DOM. Здесь нужно остановить всё, что продолжало бы работать фоном:

```vue
<script setup lang="ts">
import { ref, onMounted, onUnmounted } from "vue";

const currentTime = ref(new Date().toLocaleTimeString());
let intervalId: ReturnType<typeof setInterval>;

onMounted(() => {
  // Запускаем обновление времени каждую секунду
  intervalId = setInterval(() => {
    currentTime.value = new Date().toLocaleTimeString();
  }, 1000);
});

onUnmounted(() => {
  // Останавливаем — иначе интервал будет работать даже после удаления компонента
  clearInterval(intervalId);
});
</script>

<template>
  <p>Текущее время: {{ currentTime }}</p>
</template>
```

Что ещё нужно чистить в `onUnmounted`:

```typescript
// Подписки на события браузера
onMounted(() => {
  window.addEventListener("resize", handleResize);
});
onUnmounted(() => {
  window.removeEventListener("resize", handleResize);
});

// WebSocket соединения
onMounted(() => {
  socket = new WebSocket("wss://...");
});
onUnmounted(() => {
  socket.close();
});
```

## 📖 onUpdated

Срабатывает после каждого обновления DOM. Нужен редко — когда надо работать с DOM-элементом после его изменения:

```vue
<script setup lang="ts">
import { ref, onUpdated } from "vue";

const messages = ref<string[]>([]);
const listRef = ref<HTMLElement | null>(null);

onUpdated(() => {
  // Прокручиваем список вниз после добавления нового сообщения
  if (listRef.value) {
    listRef.value.scrollTop = listRef.value.scrollHeight;
  }
});
</script>

<template>
  <ul ref="listRef">
    <li v-for="msg in messages" :key="msg">{{ msg }}</li>
  </ul>
</template>
```

> Если нужно реагировать на изменение конкретных данных — используй `watch`, не `onUpdated`. `onUpdated` срабатывает при любом обновлении компонента.

---

## 📖 Когда что использовать

| Задача | Инструмент |
|---|---|
| Загрузить данные при открытии компонента | `onMounted` |
| Запустить таймер / подписаться на событие | `onMounted` |
| Остановить таймер / отписаться от события | `onUnmounted` |
| Прокрутить DOM-элемент после обновления | `onUpdated` |
| Реагировать на изменение конкретного `ref` | `watch` (следующий раздел) |
| Вычислить производное значение | `computed` |

## 🔧 Практика 4.1.5.1: Загрузка данных в onMounted

Сейчас `WeatherCard` показывает статичные данные. Подключим реальный OpenWeatherMap API — тот же, что использовался в TypeScript-разделе.

**Подготовка:** добавь API-ключ в `.env` в корне Vue-проекта:

```
VITE_WEATHER_API_KEY=твой_ключ_здесь
```

Vite автоматически подхватывает переменные с префиксом `VITE_`, они доступны через `import.meta.env.VITE_WEATHER_API_KEY`.

<details>
<summary>💡 Где взять API-ключ</summary>

1. Зарегистрируйся на [openweathermap.org](https://openweathermap.org/appid)
2. После входа перейди в **My API Keys** (в меню профиля)
3. Скопируй ключ из раздела **Default** или создай новый
4. Ключ активируется в течение нескольких минут после регистрации

Бесплатный план позволяет делать до 1000 запросов в день — для разработки более чем достаточно.

async function fetchWeather(city: string) {
  isLoading.value = true;
  error.value = null;

  try {
    const response = await fetch(
      `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(city)}&appid=${API_KEY}&units=metric&lang=ru`

      country: data.sys.country,

      description: data.weather[0].description,
      icon: data.weather[0].icon,

  } finally {
    isLoading.value = false;
  }
}

  fetchWeather(props.city);
});
