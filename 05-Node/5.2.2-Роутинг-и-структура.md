# 5.2.2 Роутинг и структура проекта

Один файл с маршрутами работает ровно до тех пор, пока их немного. Когда маршрутов становится 10–15, файл превращается в нечитаемую мешанину из бизнес-логики, HTTP-кода и обращений к данным. В этом разделе разберём, как разделить код так, чтобы каждый файл отвечал за одно и менять одну вещь не требовало правок в пяти местах.

## 📖 Router — разделение маршрутов

`Router` — это мини-приложение Express. Он принимает маршруты, middleware и группирует их по функциональности. В основном файле `index.ts` остаётся только подключение роутеров:

```typescript
// src/routes/weather.router.ts
import { Router } from "express";

const router = Router();

router.get("/:city", async (req, res) => {
  const { city } = req.params;
  res.json({ city });
});

export default router;
```

```typescript
// src/index.ts
import express from "express";
import weatherRouter from "./routes/weather.router";

const app = express();
app.use(express.json());

app.use("/api/weather", weatherRouter); // /api/weather/:city

app.listen(3000);
```

Теперь все маршруты погоды живут в `weather.router.ts`, а `index.ts` знает только о том, что роутер существует и подключён на `/api/weather`.

## 📖 Структура routes → controllers → services

Три слоя — стандартная структура Express-приложений. Каждый слой знает только о своих соседях:

```
routes/        — маршруты и привязка URL к обработчикам
controllers/   — обработка HTTP: читает req, формирует res
services/      — бизнес-логика: работа с данными, внешними API, расчёты
```

**Route** — только объявление маршрута, без логики:

```typescript
// src/routes/weather.router.ts
import { Router } from "express";
import { getWeather } from "../controllers/weather.controller";

const router = Router();
router.get("/:city", getWeather);

export default router;
```

**Controller** — читает данные из запроса, вызывает сервис, возвращает ответ. Не знает, откуда приходят данные:

```typescript
// src/controllers/weather.controller.ts
import { Request, Response } from "express";
import { fetchWeather } from "../services/weather.service";

export async function getWeather(
  req: Request<{ city: string }>,
  res: Response
): Promise<void> {
  try {
    const { city } = req.params;
    const weather = await fetchWeather(city);
    res.json(weather);
  } catch (error) {
    res.status(500).json({ error: "Внутренняя ошибка сервера" });
  }
}
```

**Service** — вся логика, никаких `req`/`res`. Легко тестировать и переиспользовать:

```typescript
// src/services/weather.service.ts
import { config } from "../config";

export interface WeatherData {
  city: string;
  temp: number;
  feels_like: number;
  description: string;
  humidity: number;
  wind_speed: number;
}

export async function fetchWeather(city: string): Promise<WeatherData> {
  const url = `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(city)}&appid=${config.weatherApiKey}&units=metric&lang=ru`;

  const response = await fetch(url);

  if (!response.ok) {
    const error = await response.json() as { message: string };
    throw new Error(error.message);
  }

  const data = await response.json() as any;

  return {
    city: data.name,
    temp: data.main.temp,
    feels_like: data.main.feels_like,
    description: data.weather[0].description,
    humidity: data.main.humidity,
    wind_speed: data.wind.speed,
  };
}
```

Правило: контроллер не должен знать, откуда берутся данные. Сервис не должен знать, кто его вызывает и как будет использован результат. Благодаря этому можно заменить реализацию сервиса (например, добавить кэш) без изменений в контроллере.

## 📖 REST-конвенции

REST — набор соглашений о том, как именовать маршруты и использовать HTTP-методы. Нет строгого стандарта, но есть практика, которой придерживается большинство:

**Методы:**

| Действие | Метод | Пример |
|----------|-------|--------|
| Получить список | GET | `GET /api/favorites` |
| Получить один элемент | GET | `GET /api/favorites/:id` |
| Создать | POST | `POST /api/favorites` |
| Заменить полностью | PUT | `PUT /api/favorites/:id` |
| Обновить частично | PATCH | `PATCH /api/favorites/:id` |
| Удалить | DELETE | `DELETE /api/favorites/:id` |

**Статус-коды:**

| Код | Значение |
|-----|---------|
| 200 | OK — запрос выполнен |
| 201 | Created — ресурс создан |
| 204 | No Content — успех, тела ответа нет |
| 400 | Bad Request — неверные данные от клиента |
| 401 | Unauthorized — не авторизован |
| 403 | Forbidden — нет прав |
| 404 | Not Found — ресурс не найден |
| 500 | Internal Server Error — ошибка на сервере |

**Именование маршрутов:**

```
/api/weather/:city        ✅ существительное, параметр
/api/getWeather           ❌ глагол в URL — не REST
/api/favorites            ✅ множественное число для коллекции
/api/favorite             ❌ единственное число для коллекции
/api/favorites/:id        ✅ параметр для конкретного элемента
```

## 📖 Итоговая структура проекта

```
src/
├── routes/
│   ├── weather.router.ts
│   └── favorites.router.ts
├── controllers/
│   ├── weather.controller.ts
│   └── favorites.controller.ts
├── services/
│   ├── weather.service.ts
│   └── favorites.service.ts
├── types/
│   └── weather.types.ts      # интерфейсы и типы
├── config.ts
└── index.ts
```

Типы выносят в отдельную папку, чтобы импортировать из одного места и не дублировать:

```typescript
// src/types/weather.types.ts
export interface WeatherData {
  city: string;
  temp: number;
  feels_like: number;
  description: string;
  humidity: number;
```

```typescript
const favorites: string[] = [];

export function getFavorites(): string[] {
  return favorites;
}

export function addFavorite(city: string): string[] {
  if (!favorites.includes(city)) {
    favorites.push(city);
  }
  return favorites;
}
```

```typescript
import { Request, Response } from "express";
import { getFavorites, addFavorite } from "../services/favorites.service";

export function getAll(req: Request, res: Response): void {
  res.json(getFavorites());
}

export function addCity(
  req: Request<{}, {}, { city?: string }>,
  res: Response
): void {
  const { city } = req.body;

  if (!city) {
    res.status(400).json({ error: "Поле city обязательно" });
    return;
  }

  const updated = addFavorite(city);
  res.status(201).json(updated);
}
```
