# 5.4.1 WebSockets

HTTP работает по модели «запрос — ответ»: клиент спрашивает, сервер отвечает, соединение закрывается. Чтобы узнать, не изменились ли данные, клиент должен спросить снова. Это называется поллинг — и это расточительно. WebSocket — другой протокол: он устанавливает постоянное двустороннее соединение. Сервер сам отправляет данные клиенту, когда они меняются. Именно так работают чаты, live-уведомления и биржевые котировки в реальном времени.

`socket.io` — библиотека поверх WebSocket с удобным API, автоматическим переподключением, комнатами и поддержкой TypeScript.

## 📖 WebSocket vs HTTP

| | HTTP | WebSocket |
|---|---|---|
| Соединение | Открывается на запрос, закрывается после ответа | Открывается один раз, остаётся активным |
| Инициатор | Всегда клиент | Любая из сторон |
| Когда использовать | Запрос данных, отправка форм | Чат, уведомления, live-данные |

WebSocket начинается с HTTP-запроса (`Upgrade: websocket`), а затем переключается на двусторонний канал. `socket.io` автоматически управляет этим переключением и откатывается на HTTP long-polling, если WebSocket недоступен.

## 📖 Установка и настройка

```bash
npm install socket.io
```

`socket.io` нужен только на сервере. Для TypeScript типы включены в пакет.

Важный момент: `socket.io` работает не с Express-приложением напрямую, а с HTTP-сервером Node.js. Поэтому `index.ts` нужно изменить — создать `httpServer` из `app` и передать его и в `socket.io`, и в `listen`:

```typescript
// src/index.ts
import { createServer } from "http";
import { Server } from "socket.io";
import { app } from "./app";
import { config } from "./config";
import { registerSocketHandlers } from "./socket/socket.handler";

const httpServer = createServer(app); // обёртка над Express-приложением

const io = new Server(httpServer, {
  cors: {
    origin: "*", // в production — конкретный домен
  },
});

registerSocketHandlers(io);

httpServer.listen(config.port, () => {
  console.log(`Сервер запущен: http://localhost:${config.port}`);
});
```

Express-приложение (`app.ts`) остаётся без изменений — все REST-маршруты работают как раньше через тот же порт.

## 📖 Типизация событий

В `socket.io` события передаются по имени — строками. TypeScript не знает, какие аргументы ожидать. Чтобы это исправить, `Server` принимает два дженерик-параметра с интерфейсами событий:

```typescript
// src/socket/socket.types.ts
import { WeatherData } from "../types/weather";

// Что сервер отправляет клиенту
interface ServerToClientEvents {
  weatherUpdate: (data: WeatherData) => void;
  subscribeError: (message: string) => void;
}

// Что клиент отправляет серверу
interface ClientToServerEvents {
  subscribeCity: (city: string) => void;
  unsubscribeCity: (city: string) => void;
}

export type { ServerToClientEvents, ClientToServerEvents };
```

```typescript
// src/index.ts
import { Server } from "socket.io";
import { ServerToClientEvents, ClientToServerEvents } from "./socket/socket.types";

const io = new Server<ClientToServerEvents, ServerToClientEvents>(httpServer, {
  cors: { origin: "*" },
});
```

Теперь `socket.emit("weatherUpdate", data)` — TypeScript проверит, что `data` соответствует `WeatherData`. А `socket.on("subscribeCity", handler)` — что `handler` принимает строку.

## 📖 Базовые события: connection, emit, on, disconnect

```typescript
// src/socket/socket.handler.ts
import { Server, Socket } from "socket.io";
import { ServerToClientEvents, ClientToServerEvents } from "./socket.types";

type TypedServer = Server<ClientToServerEvents, ServerToClientEvents>;
type TypedSocket = Socket<ClientToServerEvents, ServerToClientEvents>;

export function registerSocketHandlers(io: TypedServer): void {
  io.on("connection", (socket: TypedSocket) => {
    console.log(`Клиент подключился: ${socket.id}`);

    // Слушаем событие от клиента
    socket.on("subscribeCity", (city) => {
      console.log(`${socket.id} подписался на ${city}`);
      // city: string — TypeScript знает тип из интерфейса
    });

    // Когда клиент отключается
    socket.on("disconnect", () => {
      console.log(`Клиент отключился: ${socket.id}`);
    });
  });
}
```

`socket.id` — уникальный идентификатор подключения, генерируется при каждом `connection`.

## 📖 Комнаты (rooms)

Комнаты — способ сгруппировать сокеты и отправить событие сразу всем в группе. Клиент может быть в нескольких комнатах одновременно.

```typescript
// Добавить сокет в комнату
socket.join("city:Москва");

// Отправить всем в комнате (включая себя)
io.to("city:Москва").emit("weatherUpdate", data);

// Отправить всем в комнате, кроме отправителя
socket.to("city:Москва").emit("weatherUpdate", data);

// Убрать сокет из комнаты
socket.leave("city:Москва");
```

Для погодного приложения: каждый город — отдельная комната. Клиент подписывается на город → сервер добавляет его в комнату `city:Москва`. Когда погода обновляется, сервер рассылает данные всем подписчикам этой комнаты.

## 📖 Live-обновление погоды

Полная реализация: клиент подписывается на город, сервер начинает периодически запрашивать погоду и рассылать обновления всем подписчикам.

```typescript
// src/socket/socket.handler.ts
import { Server, Socket } from "socket.io";
import { ServerToClientEvents, ClientToServerEvents } from "./socket.types";
import { fetchWeather } from "../services/weather.service";
import { AppError } from "../middleware/AppError";

type TypedServer = Server<ClientToServerEvents, ServerToClientEvents>;
type TypedSocket = Socket<ClientToServerEvents, ServerToClientEvents>;

// Интервалы по городам: city → NodeJS.Timeout
const cityIntervals = new Map<string, NodeJS.Timeout>();

// Счётчик подписчиков по городам
const subscriberCount = new Map<string, number>();

const POLL_INTERVAL_MS = 30_000; // 30 секунд

async function startCityPolling(io: TypedServer, city: string): Promise<void> {
  // Запрашиваем погоду и рассылаем всем в комнате
  const poll = async () => {
    try {
      const weather = await fetchWeather(city, "metric");
      io.to(`city:${city}`).emit("weatherUpdate", weather);
    } catch (err) {
      io.to(`city:${city}`).emit("subscribeError", `Не удалось получить погоду для ${city}`);
    }
  };

  await poll(); // первый запрос сразу
  const interval = setInterval(poll, POLL_INTERVAL_MS);
  cityIntervals.set(city, interval);
}

function stopCityPolling(city: string): void {
  const interval = cityIntervals.get(city);
  if (interval) {
    clearInterval(interval);
    cityIntervals.delete(city);
  }
}

export function registerSocketHandlers(io: TypedServer): void {
  io.on("connection", (socket: TypedSocket) => {
    console.log(`Клиент подключился: ${socket.id}`);

    socket.on("subscribeCity", async (city) => {
      if (!city || typeof city !== "string") {
        socket.emit("subscribeError", "Название города не может быть пустым");
        return;
      }

      const room = `city:${city}`;
      socket.join(room);

      // Увеличиваем счётчик подписчиков
      const count = (subscriberCount.get(city) ?? 0) + 1;
      subscriberCount.set(city, count);

      // Поллинг запускается только для первого подписчика
      if (count === 1) {
        await startCityPolling(io, city);
      } else {
        // Остальные сразу получают текущие данные через отдельный запрос
        try {
          const weather = await fetchWeather(city, "metric");
          socket.emit("weatherUpdate", weather);
        } catch {
          socket.emit("subscribeError", `Не удалось получить погоду для ${city}`);
        }
      }
    });

    socket.on("unsubscribeCity", (city) => {
      const room = `city:${city}`;
      socket.leave(room);

      const count = (subscriberCount.get(city) ?? 1) - 1;
      subscriberCount.set(city, count);

      // Останавливаем поллинг, если подписчиков не осталось
      if (count <= 0) {
        subscriberCount.delete(city);
        stopCityPolling(city);
      }
    });

    socket.on("disconnect", () => {
      console.log(`Клиент отключился: ${socket.id}`);
      // socket.io автоматически убирает сокет из всех комнат при disconnect
    });
  });
}
```

## 📖 Проверка через тестовую страницу

Создай файл `test-socket.html` в корне проекта и открой его в браузере напрямую (`File → Open` или просто перетащи в окно браузера). Он подгружает клиентскую библиотеку socket.io с CDN, подключается к серверу и позволяет подписываться на города и смотреть ответы.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8" />
  <title>WebSocket тест</title>
  <script src="https://cdn.socket.io/4.8.1/socket.io.min.js"></script>
  <style>
    body { font-family: monospace; padding: 20px; }
    #log { border: 1px solid #ccc; padding: 10px; height: 300px; overflow-y: auto; }
    .error { color: red; }
    .ok { color: green; }
    input { width: 200px; margin-right: 8px; }
  </style>
</head>
<body>
  <h2>Socket.IO — тест погоды</h2>

  <div>
    <input id="cityInput" type="text" placeholder="Город, например Москва" />
    <button onclick="subscribe()">Подписаться</button>
    <button onclick="unsubscribe()">Отписаться</button>
  </div>

  <br />
  <div id="log"></div>

  <script>
    const socket = io("http://localhost:3000");
    const log = document.getElementById("log");

    function addLog(text, cls = "") {
      const line = document.createElement("div");
      line.className = cls;
      line.textContent = new Date().toLocaleTimeString() + "  " + text;
      log.appendChild(line);
      log.scrollTop = log.scrollHeight;
    }

    socket.on("connect", () => addLog("✅ Подключён: " + socket.id, "ok"));
    socket.on("disconnect", () => addLog("❌ Отключён", "error"));

    socket.on("weatherUpdate", (data) => {
      addLog(JSON.stringify(data), "ok");
    });

    socket.on("subscribeError", (msg) => {
      addLog("Ошибка: " + msg, "error");
    });

    function subscribe() {
      const city = document.getElementById("cityInput").value.trim();
      if (!city) return;
      socket.emit("subscribeCity", city);
      addLog("→ subscribeCity: " + city);
    }

    function unsubscribe() {
      const city = document.getElementById("cityInput").value.trim();
      if (!city) return;
      socket.emit("unsubscribeCity", city);
      addLog("→ unsubscribeCity: " + city);
    }
  </script>
</body>
</html>
```

Запусти сервер (`npm run dev`), открой файл в браузере — и нажми «Подписаться». Обновления погоды будут появляться в логе.

> Файл можно добавить в `.gitignore`, если не хочешь коммитить его в репозиторий.

## 🎯 Финальное задание

Добавь автоматическое периодическое обновление погоды для всех подписчиков города.

1. В `socket.handler.ts` создай `Map<string, NodeJS.Timeout>` для хранения интервалов по городам и `Map<string, number>` для счётчика подписчиков.
2. При первом подписчике города — запускай `setInterval` с интервалом 60 секунд, который запрашивает погоду и рассылает через `io.to(\`city:${city}\`).emit("weatherUpdate", data)`.
3. При каждом новом подписчике — увеличивай счётчик.
4. При `unsubscribeCity` — уменьшай счётчик. Когда счётчик доходит до нуля — вызывай `clearInterval` и удаляй интервал из `Map`.
5. Проверь: подключи двух клиентов к одному городу — оба должны получать обновления. После отключения обоих — поллинг должен прекращаться (проверь по логам в терминале).
