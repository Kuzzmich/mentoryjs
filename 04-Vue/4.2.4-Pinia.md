# 4.2.4 Pinia

`ref` и `reactive` хорошо работают когда состояние принадлежит одному компоненту. Но когда одни и те же данные нужны в разных частях дерева — их приходится передавать через props вниз и emit вверх через несколько уровней. Это называют «prop drilling», и Pinia решает эту проблему: состояние хранится глобально и доступно в любом компоненте напрямую.

---

## 🎯 Что ты узнаешь

- Когда `ref` не хватает и нужен стор
- Создание стора через `defineStore`
- `storeToRefs` — деструктуризация без потери реактивности
- Типизация стора с TypeScript
- Actions с асинхронными запросами

---

## 📖 Зачем нужен стор

Представь структуру приложения:

```
App.vue
├── HomeView.vue
│   └── CityButton.vue
└── WeatherView.vue
    ├── WeatherCard.vue
    └── ForecastList.vue
```

Список избранных городов нужен в `HomeView` (показать звёздочку) и в `WeatherCard` (кнопка добавить/удалить). Без стора придётся поднимать `favorites` в `App.vue` и передавать через props на два уровня вниз. Со стором — любой компонент просто импортирует стор и читает `favorites` напрямую.

### Стор — не замена props

Глобальное хранилище доступно везде, но это не значит, что всё нужно в него класть. Пихать в стор всё подряд — распространённая ошибка.

Хорошее правило: **состояние хранится там, где оно нужно, и не выше**.

| Ситуация | Инструмент |
|---|---|
| Данные нужны только одному компоненту | Локальный `ref` / `reactive` |
| Данные нужны родителю и одному-двум детям | Props + emit |
| Данные нужны во многих несвязанных местах | Стор |
| Данные нужны между соседними компонентами | Поднять в ближайшего общего родителя |

`unit` (°C/°F) в `WeatherCard` — хороший кандидат для локального состояния: он нужен только этому компоненту. `favorites` — хороший кандидат для стора: он нужен и на главной, и на странице погоды. Не надо тащить `unit` в Pinia только потому что Pinia есть.

Стор с десятками несвязанных переменных — такой же запах кода, как компонент на тысячу строк.

## 📖 defineStore

Стор создаётся через `defineStore`. Pinia поддерживает два синтаксиса — Options и Setup. Setup-синтаксис ближе к Composition API и лучше дружит с TypeScript:

```typescript
// src/stores/weather.ts
import { ref, computed } from "vue";
import { defineStore } from "pinia";
import type { WeatherData } from "../types/weather";

export const useWeatherStore = defineStore("weather", () => {
  // state — обычные ref
  const weatherCache = ref<Map<string, WeatherData>>(new Map());
  const selectedCity = ref("Москва");
  const unit = ref<"celsius" | "fahrenheit">("celsius");
  const isLoading = ref(false);
  const error = ref<string | null>(null);

  // getters — computed
  const currentWeather = computed(() =>
    weatherCache.value.get(selectedCity.value) ?? null
  );

  const cachedCities = computed(() =>
    Array.from(weatherCache.value.keys())
  );

  // actions — обычные функции
  const API_KEY = import.meta.env.VITE_WEATHER_API_KEY;

  async function fetchWeather(city: string) {
    if (weatherCache.value.has(city)) {
      selectedCity.value = city;
      return;
    }

    isLoading.value = true;
    error.value = null;

    try {
      const response = await fetch(
        `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(city)}&appid=${API_KEY}&units=metric&lang=ru`
      );
      if (!response.ok) throw new Error("Город не найден");
      const data = await response.json();

      weatherCache.value.set(city, {
        city: data.name,
        country: data.sys.country,
        temperature: Math.round(data.main.temp),
        feelsLike: Math.round(data.main.feels_like),
        humidity: data.main.humidity,
        description: data.weather[0].description,
        icon: data.weather[0].icon,
        timestamp: Date.now(),
      });

      selectedCity.value = city;
    } catch (e) {
      error.value = e instanceof Error ? e.message : "Ошибка загрузки";
    } finally {
      isLoading.value = false;
    }
  }

  function setUnit(newUnit: "celsius" | "fahrenheit") {
    unit.value = newUnit;
  }

  return {
    selectedCity,
    unit,
    isLoading,
    error,
    currentWeather,
    cachedCities,
    fetchWeather,
    setUnit,
  };
});
```

В Setup-синтаксе: `ref` → state, `computed` → getters, функции → actions. Возвращаешь только то, что нужно снаружи.

## 📖 Использование стора в компоненте

```vue
<script setup lang="ts">
import { useWeatherStore } from "../stores/weather";
import { storeToRefs } from "pinia";

const store = useWeatherStore();

// storeToRefs — деструктурируй state и getters с сохранением реактивности
const { selectedCity, currentWeather, isLoading, error, unit } = storeToRefs(store);

// Actions деструктурируй напрямую — они не реактивные, это просто функции
const { fetchWeather, setUnit } = store;
</script>

<template>
  <WeatherCard
    v-if="currentWeather"
    :weather-data="currentWeather"
    :unit="unit"
    @unit-change="setUnit"
  />
</template>
```

### Почему storeToRefs, а не просто деструктуризация

```typescript
// ❌ Сломает реактивность — то же что с reactive
const { selectedCity, isLoading } = store;
// selectedCity и isLoading — просто строка и boolean, не ref

// ✅ storeToRefs оборачивает каждое свойство в ref
const { selectedCity, isLoading } = storeToRefs(store);
// теперь они реактивны, шаблон обновится при изменении
```

Actions деструктурировать безопасно — они обычные функции, реактивность им не нужна.

## 📖 Стор для избранного

Отдельный стор для отдельной зоны ответственности:

```typescript
// src/stores/favorites.ts
import { ref, computed } from "vue";
import { defineStore } from "pinia";

export const useFavoritesStore = defineStore("favorites", () => {
  const cities = ref<string[]>(
    JSON.parse(localStorage.getItem("favorites") ?? "[]")
  );

  const count = computed(() => cities.value.length);

  function toggle(city: string) {
    const idx = cities.value.indexOf(city);
    if (idx === -1) {
      cities.value.push(city);
    } else {
      cities.value.splice(idx, 1);
    }
    localStorage.setItem("favorites", JSON.stringify(cities.value));
  }

  function isFavorite(city: string) {
    return cities.value.includes(city);
  }

  return { cities, count, toggle, isFavorite };
});
```
