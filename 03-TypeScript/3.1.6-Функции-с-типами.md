# 3.1.6 Функции с типами

Функции — основа любой программы. TypeScript позволяет типизировать параметры и возвращаемые значения, что делает код безопаснее и понятнее.

---

## 🎯 Что ты узнаешь

- Типизация параметров и возвращаемого значения
- Optional параметры (`?`)
- Default параметры
- Rest параметры (`...`)
- Function types (тип функции)
- Callback функции с типами
- Стрелочные функции
- Перегрузка функций (function overloads)
- `void` и `never` в функциях

---

## 📝 Базовая типизация функций

### Параметры и возврат

```typescript
function greet(name: string): string {
  return `Привет, ${name}!`;
}

const message = greet("Иван");  // Тип: string
```

**Синтаксис:**
```typescript
function имя(параметр: тип): типВозврата {
  // тело функции
}
```

---

### Множество параметров

```typescript
function calculateTemp(
  celsius: number,
  decimals: number
): string {
  return celsius.toFixed(decimals) + "°C";
}

calculateTemp(20.5678, 2);  // "20.57°C"
```

---

### Функции без возврата (void)

```typescript
function logWeather(city: string, temp: number): void {
  console.log(`${city}: ${temp}°C`);
  // Нет return или return без значения
}
```

> 💡 `void` означает что функция не возвращает значение. Можно явно вернуть `undefined` или не возвращать ничего.

---

## ❓ Optional параметры

Параметр может быть необязательным:

```typescript
function getWeather(city: string, includeHumidity?: boolean): string {
  if (includeHumidity) {
    return `${city}: 20°C, влажность 65%`;
  }
  return `${city}: 20°C`;
}

getWeather("Москва");           // ✅ Без humidity
getWeather("Казань", true);     // ✅ С humidity
```

**Важно:**
- Optional параметры помечаются `?`
- Внутри функции тип будет `boolean | undefined`
- Optional параметры должны быть **после обязательных**

---

### Проверка optional параметра

```typescript
function formatTemp(temp: number, unit?: string): string {
  // unit имеет тип: string | undefined

  if (unit === "F") {
    return `${(temp * 9/5) + 32}°F`;
  }

  // Если unit не передан или не "F"
  return `${temp}°C`;
}

formatTemp(20);         // "20°C"
formatTemp(20, "F");    // "68°F"
formatTemp(20, "C");    // "20°C"
```

---

## 🎨 Default параметры

Можно задать значение по умолчанию:

```typescript
function getWeather(city: string, unit: string = "celsius"): string {
  return `${city}: 20° (${unit})`;
}

getWeather("Москва");              // unit = "celsius"
getWeather("Казань", "fahrenheit"); // unit = "fahrenheit"
```

**Отличие от optional:**
- Optional: `unit?: string` → тип `string | undefined`
- Default: `unit: string = "celsius"` → тип `string`

---

### Default + union типы

```typescript
type Unit = "celsius" | "fahrenheit";

function convertTemp(
  temp: number,
  from: Unit = "celsius",
  to: Unit = "fahrenheit"
): number {
  if (from === to) return temp;

  if (from === "celsius") {
    return (temp * 9/5) + 32;
  }
  return (temp - 32) * 5/9;
}

convertTemp(20);                    // 20°C → °F
convertTemp(68, "fahrenheit");      // 68°F → °C
```

---

## 📦 Rest параметры

Собирает все оставшиеся аргументы в массив:

```typescript
function getAvgTemp(...temperatures: number[]): number {
  const sum = temperatures.reduce((acc, temp) => acc + temp, 0);
  return sum / temperatures.length;
}

getAvgTemp(20, 18, 22);          // 20
getAvgTemp(15, 17, 19, 21, 23);  // 19
```

**Синтаксис:**
```typescript
function имя(...параметр: тип[]): возврат {
  // параметр — это массив
}
```

---

### Комбинирование параметров

```typescript
function logTemperatures(
  city: string,           // Обязательный
  unit: string = "°C",    // Default
  ...temps: number[]      // Rest (всегда последний!)
): void {
  console.log(`${city}:`);
  temps.forEach(temp => {
    console.log(`  ${temp}${unit}`);
  });
}

logTemperatures("Москва", "°C", 20, 18, 22);
// Москва:
//   20°C
//   18°C
//   22°C
```

> ⚠️ **Порядок параметров:** обязательные → default → optional → rest

## 🔗 Function Types

Можно описать тип функции отдельно:

```typescript
// Тип функции
type Formatter = (value: number) => string;

// Функции с этим типом
const toFixed: Formatter = (value) => value.toFixed(2);
const toCelsius: Formatter = (value) => `${value}°C`;

function format(temp: number, formatter: Formatter): string {
  return formatter(temp);
}

format(20.567, toFixed);     // "20.57"
format(20, toCelsius);       // "20°C"
```

---

### Function type в параметре

```typescript
function fetchData(
  url: string,
  onSuccess: (data: any) => void,
  onError: (error: string) => void
): void {
  fetch(url)
    .then(response => response.json())
    .then(data => onSuccess(data))
    .catch(error => onError(error instanceof Error ? error.message : String(error)));
}

fetchData(
  "/api/weather",
  (data) => console.log("Успех:", data),
  (error) => console.log("Ошибка:", error)
);
```

## ➡️ Стрелочные функции

Стрелочные функции типизируются так же:

```typescript
const add = (a: number, b: number): number => {
  return a + b;
};

// Краткая форма (без фигурных скобок)
const multiply = (a: number, b: number): number => a * b;

// Без параметров
const getTimestamp = (): number => Date.now();

// Один параметр (скобки можно опустить)
const double = (n: number): number => n * 2;
```

---

### Стрелочные функции как параметры

```typescript
function processTemps(
  temps: number[],
  transform: (temp: number) => number
): number[] {
  return temps.map(transform);
}

const temps = [20, 18, 22];

// Передаём стрелочную функцию
const fahrenheit = processTemps(temps, (t) => (t * 9/5) + 32);
// [68, 64.4, 71.6]
```

---

## 🎭 Callback функции

Callback — функция, переданная как параметр:

```typescript
function loadWeather(
  city: string,
  onComplete: (temp: number) => void
): void {
  setTimeout(() => {
    const temp = Math.random() * 30;
    onComplete(temp);
  }, 1000);
}

loadWeather("Москва", (temp) => {
  console.log(`Температура: ${temp.toFixed(1)}°C`);
});
```

---

### Callback с несколькими параметрами

```typescript
type WeatherCallback = (
  city: string,
  temp: number,
  error: string | null
) => void;

function fetchWeather(city: string, callback: WeatherCallback): void {
  setTimeout(() => {
    if (Math.random() > 0.8) {
      callback(city, 0, "Ошибка сети");
    } else {
      callback(city, 20, null);
    }
  }, 1000);
}

fetchWeather("Казань", (city, temp, error) => {
  if (error) {
    console.log(`Ошибка для ${city}: ${error}`);
  } else {
    console.log(`${city}: ${temp}°C`);
  }
});
```

## 🔄 Перегрузка функций

Функция может иметь **несколько сигнатур**:

```typescript
// Сигнатуры (объявления)
function formatTemp(temp: number): string;
function formatTemp(temp: number, decimals: number): string;
function formatTemp(temp: number, unit: string): string;
function formatTemp(temp: number, unit: string, decimals: number): string;

// Реализация (одна!)
function formatTemp(
  temp: number,
  arg2?: number | string,
  arg3?: number
): string {
  let decimals = 1;
  let unit = "°C";

  if (typeof arg2 === "number") {
    decimals = arg2;
  } else if (typeof arg2 === "string") {
    unit = arg2;
    if (arg3 !== undefined) {
      decimals = arg3;
    }
  }

  return `${temp.toFixed(decimals)}${unit}`;
}

// Все варианты работают:
formatTemp(20);              // "20.0°C"
formatTemp(20, 2);           // "20.00°C"
formatTemp(20, "°F");        // "20.0°F"
formatTemp(20, "°F", 2);     // "20.00°F"
```

**Важно:**
- Сигнатуры описывают **разные способы вызова**
- Реализация **одна** и должна покрывать все случаи
- TypeScript выбирает подходящую сигнатуру при вызове

---

### Перегрузка для разных типов возврата

```typescript
function getWeatherData(city: string): string;
function getWeatherData(city: string, detailed: true): {
  city: string;
  temp: number;
  humidity: number;
};

function getWeatherData(
  city: string,
  detailed?: boolean
): string | { city: string; temp: number; humidity: number } {
  const data = {
    city,
    temp: 20,
    humidity: 65
  };

  if (detailed) {
    return data;
  }

  return `${city}: ${data.temp}°C`;
}

const simple = getWeatherData("Москва");
// Тип: string

const full = getWeatherData("Казань", true);
// Тип: { city: string; temp: number; humidity: number }
```

## ⚡ Never в функциях

Функция возвращает `never`, если **никогда не завершается**:

```typescript
function throwError(message: string): never {
  throw new Error(message);
  // Код сюда не дойдёт
}

function infiniteLoop(): never {
  while (true) {
    // Бесконечный цикл
  }
}
```

**Использование:**

```typescript
function processWeather(data: unknown): number {
  if (typeof data !== "object" || data === null) {
    throwError("Invalid data");  // never — функция не вернётся
  }

  // TypeScript знает что data — это object
  return (data as any).temperature;
}
```

## 🔧 Практика 3.1.6.2: Функции с callback

Создадим модуль валидации с callback функциями для обработки результатов.

### Шаг 0: Создай модуль валидации

Создай `src/modules/utils/validation.ts`:

```typescript
// src/modules/utils/validation.ts

export function validateCity(city, onValid, onInvalid) {
  const trimmed = city.trim();

  if (trimmed.length === 0) {
    onInvalid("Название города не может быть пустым");
    return;
  }

  if (trimmed.length < 2) {
    onInvalid("Название города слишком короткое (минимум 2 символа)");
    return;
  }

  if (!/^[а-яА-ЯёЁa-zA-Z\s-]+$/.test(trimmed)) {
    onInvalid("Название города содержит недопустимые символы");
    return;
  }

  onValid(trimmed);
}

export function validateTemperature(temp, onValid, onInvalid) {
  if (temp < -90) {
    onInvalid("Температура слишком низкая (минимум -90°C)");
    return;
  }

  if (temp > 60) {
    onInvalid("Температура слишком высокая (максимум +60°C)");
    return;
  }

  onValid(temp);
}

export function processAsync(
  value,
  validator,
  onComplete
) {
  setTimeout(() => {
    const isValid = validator(value);
    onComplete(isValid, isValid ? "Успех" : "Ошибка валидации");
  }, 100);
}
```

---

### Задания

**1. Типизируй callback функции для `validateCity`:**

   - `onInvalid` — функция, принимает сообщение об ошибке (строка), ничего не возвращает

**2. Типизируй callback функции для `validateTemperature`:**

   - `onInvalid` — функция, принимает сообщение об ошибке (строка), ничего не возвращает

**3. Создай type aliases для callback функций:**

**4. Типизируй `processAsync`:**

## 🔧 Практика 3.1.6.3: Конвертер температур с перегрузкой

Создадим функцию конвертации температур с несколькими вариантами вызова.

### Шаг 0: Создай модуль конвертера

Создай `src/modules/utils/converter.ts`:

```typescript
// src/modules/utils/converter.ts

// Здесь будут сигнатуры перегрузок

// Реализация
export function convertTemperature(temp, from, to) {
  if (from === to) {
    return temp;
  }

  if (from === "C" && to === "F") {
    return (temp * 9/5) + 32;
  }

  if (from === "F" && to === "C") {
    return (temp - 32) * 5/9;
  }

  if (from === "C" && to === "K") {
    return temp + 273.15;
  }

  if (from === "K" && to === "C") {
    return temp - 273.15;
  }

  if (from === "F" && to === "K") {
    return ((temp - 32) * 5/9) + 273.15;
  }

  if (from === "K" && to === "F") {
    return ((temp - 273.15) * 9/5) + 32;
  }

  return temp;
}
```

---

### Задания

**1. Создай type alias для единиц:**
   - `TemperatureUnit` = "C" | "F" | "K"

**2. Добавь сигнатуры перегрузок (перед реализацией):**
   - Вариант 1: `(temp: number, from: TemperatureUnit, to: TemperatureUnit) => number`
   - Вариант 2: `(temp: number, from: TemperatureUnit) => number` — конвертирует из from в Цельсии (to = "C")
   - Вариант 3: `(temp: number) => number` — предполагает что temp в Фаренгейтах, конвертирует в Цельсии

**3. Типизируй реализацию:**
   - Все параметры должны быть optional (кроме temp)
   - Используй default значения для from и to
   - Возвращает число

**4. Создай функцию для форматированного вывода:**
   - `formatConversion(temp, from, to)` — возвращает строку типа "20°C = 68°F"
   - Использует `convertTemperature` внутри
   - Типизируй все параметры

---

### Проверка

```typescript
// src/test-converter.ts
```

### Ошибка 1: Optional параметр перед обязательным

**Проблема:**
```typescript
function greet(name?: string, age: number) {  // ❌
  return `${name}, ${age} лет`;
}
```

**Решение:**
```typescript
function greet(age: number, name?: string) {  // ✅
  return name ? `${name}, ${age} лет` : `${age} лет`;
}
```

**Проблема:**
```typescript
function getTemp(city: string) {  // Возврат: any
  return 20;
}
```

**Решение:**
```typescript
function getTemp(city: string): number {  // ✅
  return 20;
}
```

### Ошибка 3: Неправильный порядок в перегрузке

**Проблема:**
```typescript
function test(a: number, b: number): number;
function test(a: number): number;
function test(a: number | string): number;  // ❌ Не покрывает все случаи

function test(a: number | string, b?: number): number {
  return typeof a === "number" ? a + (b || 0) : 0;
}
```

**Решение:**
```typescript
// Специфичные сигнатуры ПЕРЕД общими
function test(a: number, b: number): number;
function test(a: number): number;

// Реализация с самой общей сигнатурой
function test(a: number, b?: number): number {
  return a + (b || 0);
}
```

```typescript
function log(...messages: string[], prefix: string) {  // ❌
```
