# 3.2.1 Классы и модификаторы

TypeScript расширяет возможности классов ES6, добавляя типизацию, модификаторы доступа и абстрактные классы. Это позволяет создавать чистую объектно-ориентированную архитектуру с инкапсуляцией и переиспользуемым кодом.

---

## 🎯 Что ты узнаешь

- Типизация полей и методов классов
- Модификаторы доступа: `public`, `private`, `protected`
- `readonly` поля для неизменяемых данных
- Геттеры и сеттеры с валидацией
- Абстрактные классы и методы
- Статические члены класса
- Сокращённый синтаксис через constructor
- Когда использовать классы вместо функций

## 📖 Теория

### Классы в TypeScript

TypeScript расширяет возможности классов ES6, добавляя:
- **Типизацию полей и методов**
- **Модификаторы доступа** (public, private, protected)
- **Readonly поля**
- **Абстрактные классы**
- **Статические члены**

Классы помогают инкапсулировать логику и данные, делая код более структурированным и переиспользуемым.

---

## 1. Базовый синтаксис классов

```typescript
class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): string {
    return `Привет, меня зовут ${this.name}`;
  }
}

const user = new User("Алексей", 25);
console.log(user.greet()); // "Привет, меня зовут Алексей"
```

### Сокращённый синтаксис через constructor

TypeScript позволяет объявлять и инициализировать поля прямо в параметрах конструктора:

```typescript
class User {
  constructor(
    public name: string,
    public age: number
  ) {}

  greet(): string {
    return `Привет, меня зовут ${this.name}`;
  }
}
```

Это эквивалентно предыдущему примеру, но короче и читаемее.

## 2. Модификаторы доступа

TypeScript предоставляет три модификатора доступа для контроля видимости членов класса:

### `public` (по умолчанию)

Доступно везде:

```typescript
class User {
  public name: string; // явное указание (можно опустить)

  constructor(name: string) {
    this.name = name;
  }
}

const user = new User("Алексей");
console.log(user.name); // ✅ "Алексей"
```

### `private`

Доступно **только внутри класса**:

```typescript
class BankAccount {
  private balance: number = 0;

  deposit(amount: number): void {
    this.balance += amount;
  }

  getBalance(): number {
    return this.balance;
  }
}

const account = new BankAccount();
account.deposit(1000);
console.log(account.getBalance()); // ✅ 1000
console.log(account.balance); // ❌ Error: Property 'balance' is private
```

### `protected`

Доступно **внутри класса и в наследниках**:

```typescript
class Animal {
  protected name: string;

  constructor(name: string) {
    this.name = name;
  }
}

class Dog extends Animal {
  bark(): void {
    console.log(`${this.name} лает`); // ✅ Доступ есть
  }
}

const dog = new Dog("Бобик");
dog.bark(); // "Бобик лает"
console.log(dog.name); // ❌ Error: Property 'name' is protected
```

## 3. Readonly поля

Поля с модификатором `readonly` можно установить только:
- При объявлении
- В конструкторе

После инициализации их **нельзя изменить**:

```typescript
class Config {
  readonly apiUrl: string = "https://api.example.com";
  readonly timeout: number;

  constructor(timeout: number) {
    this.timeout = timeout;
  }

  updateUrl(newUrl: string): void {
    this.apiUrl = newUrl; // ❌ Error: Cannot assign to 'apiUrl'
  }
}

const config = new Config(5000);
config.timeout = 3000; // ❌ Error: Cannot assign to 'timeout'
```

### Readonly + private

Можно комбинировать модификаторы:

```typescript
class ApiService {
  private readonly apiKey: string;

  constructor(apiKey: string) {
    this.apiKey = apiKey;
  }

  // apiKey нельзя изменить даже внутри класса (после конструктора)
}
```

## 4. Геттеры и сеттеры

TypeScript поддерживает `get` и `set` для контролируемого доступа к полям:

```typescript
class Temperature {
  private _celsius: number = 0;

  get celsius(): number {
    return this._celsius;
  }

  set celsius(value: number) {
    if (value < -273.15) {
      throw new Error("Температура не может быть ниже абсолютного нуля");
    }
    this._celsius = value;
  }

  get fahrenheit(): number {
    return this._celsius * 9/5 + 32;
  }

  set fahrenheit(value: number) {
    this.celsius = (value - 32) * 5/9;
  }
}

const temp = new Temperature();
temp.celsius = 25;
console.log(temp.fahrenheit); // 77

temp.fahrenheit = 32;
console.log(temp.celsius); // 0

temp.celsius = -300; // ❌ Error: Температура не может быть ниже...
```

## 5. Абстрактные классы

Абстрактные классы:
- **Нельзя создать напрямую** (только через наследников)
- Могут содержать **абстрактные методы** (без реализации)
- Наследники **обязаны реализовать** абстрактные методы

```typescript
abstract class Shape {
  constructor(public name: string) {}

  // Абстрактный метод (нет реализации)
  abstract getArea(): number;

  // Обычный метод (есть реализация)
  describe(): string {
    return `Это ${this.name} с площадью ${this.getArea()}`;
  }
}

class Circle extends Shape {
  constructor(public radius: number) {
    super("круг");
  }

  getArea(): number {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(
    public width: number,
    public height: number
  ) {
    super("прямоугольник");
  }

  getArea(): number {
    return this.width * this.height;
  }
}

const circle = new Circle(5);
console.log(circle.describe()); // "Это круг с площадью 78.54"

const rect = new Rectangle(4, 6);
console.log(rect.describe()); // "Это прямоугольник с площадью 24"

const shape = new Shape("фигура"); // ❌ Error: Cannot create an instance of an abstract class
```

## 6. Статические члены

Статические поля и методы принадлежат **классу**, а не экземпляру:

```typescript
class MathUtils {
  static PI: number = 3.14159;

  static calculateCircleArea(radius: number): number {
    return this.PI * radius ** 2;
  }

  static max(...numbers: number[]): number {
    return Math.max(...numbers);
  }
}

console.log(MathUtils.PI); // 3.14159
console.log(MathUtils.calculateCircleArea(5)); // 78.54
console.log(MathUtils.max(1, 5, 3, 9, 2)); // 9

const utils = new MathUtils();
console.log(utils.PI); // ❌ Error: Property 'PI' is a static member
```

### Статические + приватные

```typescript
class Counter {
  private static count: number = 0;

  constructor() {
    Counter.count++;
  }

  static getCount(): number {
    return Counter.count;
  }
}

const c1 = new Counter();
const c2 = new Counter();
const c3 = new Counter();

console.log(Counter.getCount()); // 3
console.log(Counter.count); // ❌ Error: Property 'count' is private
```

## 7. Когда использовать классы

### ✅ Используй классы когда:
- Нужна **инкапсуляция** (скрыть внутреннюю реализацию)
- Есть **состояние**, которым нужно управлять
- Логика **сложная** и требует структурирования
- Нужны **наследование** и **полиморфизм**

### ❌ Не используй классы когда:
- Достаточно простых функций
- Нет состояния (stateless логика)
- Работаешь в функциональном стиле

function displayWeather(data: WeatherData, fromCache: boolean): void {
  errorDiv.style.display = "none";
  loadingDiv.style.display = "none";
  weatherCard.style.display = "block";

  const unit = weatherService.units;

  cityNameEl.textContent = `${data.city}, ${data.country}`;
  descriptionEl.textContent = data.description;

  temperatureEl.textContent = WeatherService.formatTemperature(data.temperature, unit);
  feelsLikeEl.textContent = WeatherService.formatTemperature(data.feelsLike, unit);
  humidityEl.textContent = `${data.humidity}%`;
  pressureEl.textContent = `${data.pressure} hPa`;
  windEl.textContent = `${data.windSpeed} м/с`;

  cacheIndicator.style.display = fromCache ? "block" : "none";
}

function showError(message: string): void {
  errorDiv.textContent = message;
  errorDiv.style.display = "block";
  weatherCard.style.display = "none";
  loadingDiv.style.display = "none";
}

async function fetchWeather(): Promise<void> {
  const city = cityInput.value.trim();

  if (!WeatherService.isValidCityName(city)) {
    showError("Введите корректное название города (минимум 2 буквы)");
    return;
  }

  errorDiv.style.display = "none";
  weatherCard.style.display = "none";
  loadingDiv.style.display = "block";

  try {
    const startTime = Date.now();
    const data = await weatherService.getData(city);
    const endTime = Date.now();

    const fromCache = endTime - startTime < 100;

    displayWeather(data, fromCache);
  } catch (error) {
    showError(error instanceof Error ? error.message : "Не удалось загрузить погоду");
  }
}
