# 3.1.3 Бандлеры и Vite

В предыдущей теме мы настроили TypeScript с чистой компиляцией. Но в реальных проектах часто используют **бандлеры** — инструменты, которые упрощают разработку и оптимизируют код для production.

---

## 🎯 Что ты узнаешь

- Что такое бандлер и зачем он нужен
- Разницу между чистым TypeScript и бандлером
- Популярные бандлеры: Webpack, Vite, Rollup, esbuild, Parcel
- Как настроить Vite для TypeScript проекта
- Преимущества Hot Module Replacement (HMR)
- Как работать без расширений `.js` в импортах
- Оптимизацию для production

---

## 📖 Что такое бандлер?

**Бандлер (Module Bundler)** — инструмент, который:
- Объединяет множество файлов в один или несколько бандлов
- Компилирует TypeScript, JSX, SCSS и другие форматы
- Оптимизирует код для production (минификация, tree-shaking)
- Предоставляет dev-сервер с горячей перезагрузкой
- Автоматически разрешает импорты (не нужны расширения `.js`)

---

## 🔄 Чистый TypeScript vs Бандлер

### Подход 1: Чистый TypeScript (как в 3.1.2)

```
src/app.ts  →  tsc  →  dist/app.js  →  Браузер
```

**Плюсы:**
- ✅ Простая настройка
- ✅ Быстрая компиляция
- ✅ Понятный процесс (TS → JS)

**Минусы:**
- ❌ Нужны расширения `.js` в импортах
- ❌ Нет оптимизации для production
- ❌ Медленная перезагрузка в браузере
- ❌ Нет Hot Module Replacement

### Подход 2: TypeScript + Бандлер

```
src/  →  Vite/Webpack  →  dist/bundle.js  →  Браузер
      (компиляция + оптимизация)
```

**Плюсы:**
- ✅ Импорты без расширений
- ✅ Hot Module Replacement (HMR)
- ✅ Автоматическая оптимизация
- ✅ Минификация, tree-shaking
- ✅ CSS, images, assets из коробки

**Минусы:**
- ❌ Сложнее настройка (но Vite упрощает)
- ❌ Ещё один инструмент в стеке

---

## 📦 Популярные бандлеры

### 1. Webpack

**Самый популярный** и мощный бандлер.

**Плюсы:**
- Огромная экосистема плагинов
- Гибкая настройка
- Поддержка всего: TS, JSX, SCSS, images, fonts

**Минусы:**
- Сложная конфигурация
- Медленный в больших проектах
- Долгий старт dev-сервера

**Когда использовать:** Большие enterprise-проекты с кастомными требованиями.

---

### 2. Vite ⚡ (рекомендуется)

**Современный** и **быстрый** инструмент от создателя Vue.js.

**Плюсы:**
- Мгновенный старт dev-сервера
- Очень быстрый HMR
- Минимальная конфигурация
- TypeScript из коробки
- Оптимизированный production build (использует Rollup)

**Минусы:**
- Относительно новый (меньше плагинов чем у Webpack)

**Когда использовать:** Большинство современных проектов (Vue, React, vanilla TS/JS).

**Популярность:** Используется в Vue 3, Nuxt 3, SvelteKit, Astro.

---

### 3. Rollup

**Специализируется** на создании библиотек.

**Плюсы:**
- Отличный tree-shaking
- Чистый output код
- Идеален для библиотек

**Минусы:**
- Не заточен под приложения
- Требует больше настройки для dev-сервера

**Когда использовать:** Создание npm-библиотек.

---

### 4. esbuild

**Самый быстрый** бандлер (написан на Go).

**Плюсы:**
- Экстремально быстрая компиляция
- Минификация из коробки

**Минусы:**
- Меньше возможностей чем Webpack
- Молодая экосистема

**Когда использовать:** Проекты где важна скорость сборки.

---

### 5. Parcel

**"Zero-config"** бандлер.

**Плюсы:**
- Нулевая конфигурация
- Всё работает из коробки

**Минусы:**
- Меньше контроля над сборкой
- Медленнее Vite

**Когда использовать:** Прототипы, обучение, простые проекты.

---

## 🎯 Какой бандлер выбрать?

### Для обучения и большинства проектов:
**→ Vite** — быстрый, простой, современный

### Для enterprise и сложных кастомизаций:
**→ Webpack** — максимум возможностей

### Для npm-библиотек:
**→ Rollup** — чистый output

### Для максимальной скорости:
**→ esbuild** — самый быстрый

## 🔧 Практика 3.1.3.1: Миграция на Vite

Возьмём проект из **практики 3.1.2.2** и переведём его на Vite.

> 💡 **Важно:** В Vite используется стандартная структура, где `index.html` находится в **корне проекта**, а не в папке `public/`. Папка `public/` используется только для статических файлов (CSS, images, fonts).

### Исходная структура (из 3.1.2.2)

```
weather-app/
├── index.html              # HTML в корне проекта
├── public/                 # Статические файлы (CSS, images)
│   └── styles/
│       └── main.css
├── src/                    # TypeScript исходники
│   ├── modules/
│   │   ├── api/
│   │   │   ├── weather.ts
│   │   │   └── geocoding.ts
│   │   ├── ui/
│   │   │   ├── weatherCard.ts
│   │   │   ├── forecast.ts
│   │   │   └── favorites.ts
│   │   ├── utils/
│   │   │   ├── format.ts
│   │   │   └── storage.ts
│   │   └── index.ts
│   ├── themes/
│   │   ├── light.ts
│   │   └── dark.ts
│   └── app.ts
├── dist/                   # Скомпилированный JS (генерируется tsc)
├── tsconfig.json
├── package.json
└── .gitignore
```

**Проблемы текущего подхода:**
- Нужны расширения `.js` в импортах
- Нет HMR (нужно обновлять браузер вручную)
- HTML копируется вручную скриптом
- Нет оптимизации для production

---

### Шаг 1: Установка Vite

```bash
# Перейди в папку проекта
cd weather-app

# Установи Vite
npm install --save-dev vite
```

---

### Шаг 2: Создание vite.config.ts

Создай файл `vite.config.ts` в корне проекта:

```typescript
import { defineConfig } from 'vite';

export default defineConfig({
  // Настройки сборки
  build: {
    outDir: './dist',           // Папка для production
    emptyOutDir: true           // Очищать dist перед сборкой
  },

  // Настройки dev-сервера
  server: {
    port: 3000,                 // Порт dev-сервера
    open: true                  // Автооткрытие браузера
  },

  // Разрешение модулей
  resolve: {
    extensions: ['.ts', '.js']  // Автоматически подставляет расширения
  }
});
```

> 💡 Vite автоматически находит `index.html` в корне проекта - никаких дополнительных настроек не нужно!

---

### Шаг 3: Обновление package.json

Замени скрипты:

```json
{
  "scripts": {
    "dev": "vite",                    // Dev-сервер с HMR
    "build": "vite build",            // Production сборка
    "preview": "vite preview",        // Просмотр production сборки
    "check": "tsc --noEmit"          // Проверка типов без компиляции
  },
  "devDependencies": {
    "typescript": "^5.3.0",
    "vite": "^5.0.0"
  }
}
```

**Что изменилось:**
- ❌ Убрали `tsc`, `live-server`, `copy:html` — Vite делает всё сам
- ✅ `dev` — запуск с HMR
- ✅ `build` — оптимизированная сборка
- ✅ `preview` — проверка production сборки локально

---

### Шаг 4: ⚠️ ВАЖНО: Убираем расширения .js из импортов

Vite автоматически разрешает импорты, поэтому `.js` больше не нужны!

**Было (с чистым TypeScript):**
```typescript
// src/app.ts
import { fetchWeather } from './modules/api.js';  // ❌ Убираем .js
import { renderWeather } from './modules/ui.js';  // ❌ Убираем .js
```

**Стало (с Vite):**
```typescript
// src/app.ts
import { fetchWeather } from './modules/api';     // ✅ Без расширения
import { renderWeather } from './modules/ui';     // ✅ Без расширения
```

**Действие:** Открой все `.ts` файлы и удали `.js` из импортов.

---

### Шаг 5: Обновление index.html

Vite требует, чтобы HTML подключал TypeScript напрямую (не скомпилированный JS):

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Погода - Vite + TypeScript</title>
  <!-- ✅ CSS подключается как обычно — Vite раздаёт папку public/ статически -->
  <link rel="stylesheet" href="/styles/main.css">
</head>
<body>
  <div class="weather-card">
    <div class="search">
      <input type="text" id="cityInput" placeholder="Введите город...">
      <button id="searchBtn">Поиск</button>
    </div>
    <div id="weatherInfo" class="weather-info"></div>
  </div>

  <!-- ⚡ Vite компилирует TypeScript на лету -->
  <script type="module" src="/src/app.ts"></script>
</body>
</html>
```

**Что изменилось:**
- ✅ Подключаем `/src/app.ts` напрямую (не `dist/app.js`)
- ✅ Используем абсолютный путь от корня проекта
- ✅ Vite компилирует TypeScript автоматически
- ✅ CSS подключается через `<link>` — Vite раздаёт `public/` как статику, путь `/styles/main.css` → `public/styles/main.css`

---

### Шаг 6: Можно удалить (опционально)

С Vite эти файлы/настройки больше не нужны:

```bash
# Можно удалить dist/ (будет генериться заново)
rm -rf dist/

# Можно упростить tsconfig.json (убрать outDir)
```

**Упрощённый tsconfig.json:**

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ES2020",
    "strict": false,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "moduleResolution": "node"
  },
  "include": ["src/**/*"]
}
```

> 💡 Убрали `outDir` и `rootDir`, потому что компиляцией теперь занимается Vite.

## ❌ Частые проблемы

### Проблема 1: "Cannot find module"

**Симптом:**
```
[vite] Internal server error: Cannot find module './modules/api'
```

**Причина:** Неправильный путь импорта.

**Решение:** Проверь относительные пути:
```typescript
// ✅ Правильно
import { api } from './modules/api';  // Находится в той же папке

// ❌ Неправильно
import { api } from 'modules/api';    // Vite ищет в node_modules
```

---

### Проблема 2: Порт 3000 занят

**Симптом:**
```
Port 3000 is in use
```

**Решение:** Измени порт в `vite.config.ts`:
```typescript
export default defineConfig({
  server: {
    port: 3001  // Или любой другой
  }
});
```

---

### Проблема 3: TypeScript ошибки не видны

**Симптом:** Vite запускается, но есть ошибки типов.

**Решение:** Vite не проверяет типы! Запускай проверку отдельно:
```bash
npm run check  # tsc --noEmit
```

Или используй плагин:
```bash
npm install -D vite-plugin-checker
```

---

### Проблема 4: fsevents.watch is not a function (macOS)

**Симптом:**
```
TypeError: fsevents.watch is not a function
```

**Причина:** Несовместимость версии модуля `fsevents`.

**Решение:**
```bash
npm upgrade fsevents
npm run dev
```

## 🔧 Практика 3.1.3.2: Добавление CSS в Vite

Vite автоматически обрабатывает CSS! Попробуем.

**Шаг 1:** Создай `src/styles/main.css`:

```css
/* src/styles/main.css */
:root {
  --primary: #667eea;
  --secondary: #764ba2;
}

body {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  margin: 0;
  padding: 20px;
  background: linear-gradient(135deg, var(--primary) 0%, var(--secondary) 100%);
  min-height: 100vh;
}

.weather-card {
  max-width: 500px;
  margin: 0 auto;
  background: white;
  padding: 30px;
  border-radius: 16px;
  box-shadow: 0 10px 30px rgba(0, 0, 0, 0.2);
}
```

**Шаг 2:** Импортируй CSS в `src/app.ts`:

```typescript
// src/app.ts
import './styles/main.css';  // ✅ Vite автоматически обработает

import { fetchWeather } from './modules/api';
import { renderWeather } from './modules/ui';

// Остальной код...
```

**Шаг 3:** Запусти `npm run dev` — стили применятся!

**Что произошло:**
- Vite добавил CSS в `<style>` тег в dev-режиме
- При сборке (`npm run build`) CSS извлечётся в отдельный файл
