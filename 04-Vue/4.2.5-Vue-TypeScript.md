# 4.2.5 Vue + TypeScript углублённо

Базовые `defineProps<T>()` и `defineEmits<T>()` ты уже знаешь. Здесь разбираем то, что встречается в реальных проектах и поначалу вызывает вопросы: типизированные ссылки на DOM и дочерние компоненты, безопасная передача данных через `provide`/`inject`, обобщённые компоненты.

---

## 🎯 Что ты узнаешь

- Типизация template refs — ссылки на DOM-элементы и компоненты
- `defineExpose` — публичный интерфейс компонента
- `provide` / `inject` с `InjectionKey<T>` — типобезопасный контекст
- Generic-компоненты (`generic="T"`)
- Правильная типизация composables

## 📖 Template refs с TypeScript

`ref` в шаблоне — это ссылка на DOM-элемент или экземпляр компонента. Без TypeScript это работает, но тип будет `any`. С TypeScript нужно явно указать тип.

### Ссылка на DOM-элемент

```vue
<script setup lang="ts">
import { ref, onMounted } from "vue";

// null — начальное значение до монтирования
const inputEl = ref<HTMLInputElement | null>(null);

onMounted(() => {
  inputEl.value?.focus(); // optional chaining — элемент может быть null
});
</script>

<template>
  <input ref="inputEl" type="text" />
</template>
```

Правило: тип всегда `T | null`, потому что до монтирования компонента `ref` равен `null`.

### Ссылка на дочерний компонент

```vue
<script setup lang="ts">
import { ref } from "vue";
import WeatherCard from "./WeatherCard.vue";

// InstanceType<typeof Component> — тип экземпляра компонента
const cardRef = ref<InstanceType<typeof WeatherCard> | null>(null);

function refreshCard() {
  cardRef.value?.refresh(); // метод, объявленный через defineExpose
}
</script>

<template>
  <WeatherCard ref="cardRef" city="Москва" />
  <button @click="refreshCard">Обновить</button>
</template>
```

`InstanceType<typeof WeatherCard>` — это утилитарный тип TypeScript, который говорит: «возьми тип конструктора компонента и дай мне тип его экземпляра». Без этого TypeScript не знает, какие методы и свойства есть у компонента.

## 📖 defineExpose — публичный интерфейс компонента

По умолчанию всё, что объявлено в `<script setup>`, **закрыто** для родителя — он не может получить доступ к внутренним `ref` и функциям через `ref`.

`defineExpose` явно объявляет то, что родитель может вызвать:

```vue
<!-- WeatherCard.vue -->
<script setup lang="ts">
import { ref } from "vue";

const isLoading = ref(false);

async function refresh() {
  isLoading.value = true;
  // ... запрос
  isLoading.value = false;
}

// Только это видно снаружи
defineExpose({ refresh });
</script>
```

Теперь родитель, у которого есть `ref` на этот компонент, может вызвать `cardRef.value?.refresh()`.

> **Не злоупотребляй `defineExpose`**. Если компонент слишком часто управляется снаружи — это признак, что логика вынесена не туда. Для большинства случаев хватает props + emit.

## 📖 provide / inject с InjectionKey

`provide`/`inject` — альтернатива prop drilling для данных, которые нужны глубоко в дереве компонентов, но не нужны в сторе (потому что они принадлежат конкретному поддереву, а не всему приложению).

**Без типизации** inject возвращает `unknown` — не очень полезно:

```typescript
// В дочернем компоненте
const theme = inject("theme"); // тип: unknown
```

**С `InjectionKey`** — тип пробрасывается автоматически:

```typescript
// keys.ts — файл с ключами, чтобы не повторяться
import type { InjectionKey, Ref } from "vue";

export const themeKey = Symbol("theme") as InjectionKey<Ref<"light" | "dark">>;
export const unitKey = Symbol("unit") as InjectionKey<Ref<"metric" | "imperial">>;
```

```vue
<!-- App.vue — провайдер -->
<script setup lang="ts">
import { ref, provide } from "vue";
import { unitKey } from "./keys";

const unit = ref<"metric" | "imperial">("metric");
provide(unitKey, unit);
</script>
```

```vue
<!-- WeatherCard.vue — потребитель -->
<script setup lang="ts">
import { inject } from "vue";
import { unitKey } from "./keys";

// Тип автоматически: Ref<"metric" | "imperial"> | undefined
const unit = inject(unitKey);

// Или с дефолтным значением (тип: Ref<"metric" | "imperial">)
const unit = inject(unitKey, ref("metric"));
</script>
```

`Symbol()` гарантирует уникальность ключа — два разных `Symbol("unit")` не равны друг другу, поэтому случайных коллизий не будет.

### provide/inject vs Pinia

| Сценарий | Инструмент |
|---|---|
| Данные нужны только в одном поддереве | `provide`/`inject` |
| Данные нужны во всём приложении | Pinia |
| Нужна персистентность (localStorage) | Pinia |
| Библиотека компонентов (без Pinia) | `provide`/`inject` |

## 📖 Generic-компоненты

Иногда нужен компонент, который работает с разными типами данных, но остаётся типобезопасным. Например, список выбора элементов: `SelectList<City>`, `SelectList<string>`, `SelectList<User>`.

```vue
<!-- SelectList.vue -->
<script setup lang="ts" generic="T">
defineProps<{
  items: T[];
  selected: T | null;
  getLabel: (item: T) => string;
}>();

defineEmits<{
  select: [item: T];
}>();
</script>

<template>
  <ul>
    <li
      v-for="(item, i) in items"
      :key="i"
      :class="{ active: item === selected }"
      @click="$emit('select', item)"
    >
      {{ getLabel(item) }}
    </li>
  </ul>
</template>
```

Использование:

```vue
<SelectList
  :items="cities"
  :selected="selectedCity"
  :get-label="(c) => c.name"
  @select="selectedCity = $event"
/>
```

TypeScript автоматически выведет `T = City` из переданного `items` и проверит, что `getLabel` принимает `City`.

> `generic="T"` — это атрибут тега `<script setup>`, не TypeScript-синтаксис внутри скрипта. Он доступен с Vue 3.3+.

Можно задать ограничения:

```vue
<script setup lang="ts" generic="T extends { id: number | string }">
```

## 📖 Типизация composables

Хорошо типизированный composable не требует аннотаций при вызове — типы выводятся автоматически.

### Принимать ref или значение — `MaybeRefOrGetter`

Если composable принимает `city` — это может быть строка, `ref<string>` или `computed`. Vue предоставляет утилитарный тип для этого:

```typescript
import { toValue, type MaybeRefOrGetter } from "vue";

export function useWeather(city: MaybeRefOrGetter<string>) {
  // toValue() разворачивает: ref.value, computed.value или просто значение
  const cityValue = toValue(city);
}
```

### Явный возвращаемый тип

Для публичного composable (используется в нескольких местах) лучше явно указать возвращаемый тип:

```typescript
interface UseWeatherReturn {
  weather: Ref<WeatherData | null>;
  isLoading: Ref<boolean>;
  error: Ref<string | null>;
  refresh: () => Promise<void>;
}

export function useWeather(city: MaybeRefOrGetter<string>): UseWeatherReturn {
  // ...
}
```

Это защищает от случайного изменения публичного API и улучшает автодополнение.

## 🎯 Финальное задание

Создай обобщённый компонент `SelectList.vue`, который работает с любым типом данных.

**Требования:**

1. `<script setup lang="ts" generic="T">` — параметр типа `T`
2. Props:
   - `items: T[]` — список элементов
   - `selected: T | null` — текущий выбранный элемент
   - `getLabel: (item: T) => string` — функция получения строки для отображения
   - `getKey: (item: T) => string | number` — функция получения ключа для `v-for`
3. Emit: `select: [item: T]`
4. Активный элемент подсвечивается классом `.active`
5. Используй компонент в `HomeView.vue` для списка городов — убедись, что TypeScript правильно выводит тип `T` без явных аннотаций

**Проверка:** `getLabel` с неверным типом аргумента должна давать ошибку TypeScript.
