# 4.1.6 Watchers

`computed` вычисляет производное значение. `watch` и `watchEffect` реагируют на изменения и запускают побочные эффекты — запросы, логирование, синхронизацию с внешними системами.

---

## 🎯 Что ты узнаешь

- `watch` — явное отслеживание конкретного источника
- `watchEffect` — автоматическое отслеживание всех зависимостей
- Когда использовать `watch`, а когда `computed` или `watchEffect`
- Как остановить watcher и избежать утечек

---

## 📖 watch

`watch` следит за конкретным источником и вызывает коллбэк когда значение меняется:

```vue
<script setup lang="ts">
import { ref, watch } from "vue";

const city = ref("Москва");
const temperature = ref(20);

// Следим за одним ref
watch(city, (newCity, oldCity) => {
  console.log(`Город изменился: ${oldCity} → ${newCity}`);
  // Здесь можно загрузить погоду для нового города
});

// Следим за несколькими источниками сразу
watch([city, temperature], ([newCity, newTemp], [oldCity, oldTemp]) => {
  console.log(`${newCity}: ${newTemp}°C`);
});
</script>
```

### Опции watch

```typescript
watch(city, async (newCity) => {
  await fetchWeather(newCity);
}, {
  immediate: true,  // запустить сразу при создании, не дожидаясь изменения
  deep: true,       // следить за вложенными свойствами объекта
});
```

`immediate: true` — полезная замена вызову в `onMounted`, когда логика одинакова:

```typescript
// Вместо двух блоков:
onMounted(() => fetchWeather(city.value));
watch(city, fetchWeather);

// Можно написать один:
watch(city, fetchWeather, { immediate: true });
```

### Слежка за объектами — важные особенности

Поведение `watch` зависит от того, как хранится объект.

**`ref` с объектом внутри — нужен `{ deep: true }` или геттер:**

```typescript
const weather = ref({ temperature: 20, humidity: 65 });

// ❌ Не сработает при изменении поля — watch видит только замену самого ref
watch(weather, (val) => console.log(val));

// ✅ Геттер на конкретное поле
watch(() => weather.value.temperature, (temp) => {
  console.log("Температура:", temp);
});

// ✅ Весь объект с deep
watch(weather, (val) => {
  console.log("Что-то изменилось:", val);
}, { deep: true });
```

**`reactive`-объект — автоматически deep:**

```typescript
const filters = reactive({ city: "Москва", unit: "celsius" });

// ✅ reactive без { deep: true } — Vue уже отслеживает вложенные свойства
watch(filters, (newFilters) => {
  console.log(newFilters);
});

// Но newFilters и oldFilters будут одним и тем же объектом!
// Чтобы сравнить старое и новое — используй геттер на конкретное поле:
watch(() => filters.city, (newCity, oldCity) => {
  console.log(`${oldCity} → ${newCity}`);
});
```

> При слежке за `reactive`-объектом целиком `oldValue` и `newValue` в коллбэке указывают на один и тот же объект — потому что Vue не делает копию. Если нужно сравнить старое и новое значение, следи за конкретным полем через геттер `() => obj.field`.

## 📖 watchEffect

`watchEffect` запускает коллбэк немедленно и автоматически отслеживает все реактивные значения, к которым он обращается:

```vue
<script setup lang="ts">
import { ref, watchEffect } from "vue";

const city = ref("Москва");
const unit = ref("celsius");

// Запустится сразу и при каждом изменении city или unit
watchEffect(async () => {
  console.log(`Загружаем: ${city.value} в ${unit.value}`);
  await fetchWeather(city.value);
});
</script>
```

Не нужно явно указывать зависимости — Vue сам определяет, что читается внутри коллбэка.

### Остановка watcher

Оба `watch` и `watchEffect` возвращают функцию остановки. В `<script setup>` они останавливаются автоматически при уничтожении компонента — явная остановка нужна только в особых случаях:

```typescript
const stop = watchEffect(() => {
  console.log(city.value);
});

// Остановить вручную — например, после первого успешного результата
function handleSuccess() {
  stop();
}
```

## 📖 Когда что использовать

| Задача | Инструмент |
|---|---|
| Вычислить значение на основе других данных | `computed` |
| Загрузить данные при изменении параметра | `watch` |
| Выполнить код при изменении нескольких источников | `watch([a, b], ...)` |
| Синхронизироваться с внешней системой, зависимости неочевидны | `watchEffect` |
| Нужны старое и новое значение | `watch` — он передаёт оба |
| Нужно запустить сразу и при изменениях | `watch(..., { immediate: true })` или `watchEffect` |

Главное отличие `watch` от `watchEffect`: `watch` ленивый (не запускается сразу) и явный (ты указываешь источник). `watchEffect` жадный (сразу) и неявный (зависимости определяются автоматически).

## 🔧 Практика 4.1.6.1: Перезагрузка данных при смене города

Сейчас `WeatherCard` загружает данные только один раз в `onMounted`. При переключении города в `App.vue` карточка пересоздаётся (благодаря `:key`), что вызывает новый `onMounted`. Добавим `watch` прямо в `WeatherCard` — это более явный подход.

**Файл для изменения:** `src/components/WeatherCard.vue`

**Стартовый код — добавления в `<script setup>`:**

```typescript
import { ref, computed, watch, onMounted, onUnmounted } from "vue";

// TODO: добавь watch за props.city
// При изменении города — вызывай fetchWeather с новым значением
// Используй геттер () => props.city как источник
```

**Ожидаемый результат:** выбираешь другой город в списке — карточка сразу начинает загрузку и показывает данные для нового города без перезагрузки страницы.

<details>
<summary>💡 Подсказка</summary>

```typescript
// Следим за props.city через геттер — props не ref, поэтому нужен () =>
watch(() => props.city, (newCity) => {
  fetchWeather(newCity);
});
```

> `onMounted` при этом можно оставить — он загружает данные при первом открытии. `watch` перезагружает при смене города.
</details>

## 🔧 Практика 4.1.6.2: Автосохранение списка городов

Добавим в `App.vue` автосохранение списка городов в `localStorage` при каждом изменении.

**Файл для изменения:** `src/App.vue`

**Стартовый код:**

```typescript
import { ref, reactive, computed, watch } from "vue";

// Загружаем сохранённый список при старте
const savedCities = localStorage.getItem("cities");
const cities = ref<string[]>(
  savedCities ? JSON.parse(savedCities) : ["Москва", "Санкт-Петербург", "Казань", "Новосибирск", "Екатеринбург"]
);

// TODO: добавь watch за cities
// При каждом изменении — сохраняй в localStorage через JSON.stringify
// Используй { deep: true } — cities это массив
```

**Ожидаемый результат:** добавь новый город, обнови страницу — он остаётся в списке. Удали город, обнови — его нет.

<details>
<summary>💡 Подсказка</summary>

```typescript
watch(cities, (newCities) => {
  localStorage.setItem("cities", JSON.stringify(newCities));
}, { deep: true });
```
</details>
