# 3.1.5 Интерфейсы vs Type aliases

В TypeScript есть два способа описать форму объекта: **интерфейсы** (interfaces) и **type aliases**. Оба решают похожие задачи, но имеют важные различия.

---

## 🎯 Что ты узнаешь

- Как объявлять интерфейсы и type aliases
- Ключевые различия между `interface` и `type`
- Когда использовать interface, а когда type
- Расширение интерфейсов через `extends`
- Optional и readonly свойства
- Индексные сигнатуры для динамических ключей
- Пересечение (intersection) типов

---

## 📋 Интерфейсы (Interfaces)

Интерфейс описывает **структуру объекта** — какие свойства и методы он должен содержать.

### Базовый синтаксис

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const user: User = {
  id: 1,
  name: "Иван",
  email: "ivan@example.com"
};
```

**Преимущества:**
- ✅ Читабельный синтаксис для объектов
- ✅ Можно расширять через `extends`
- ✅ Можно объединять (declaration merging)
- ✅ Лучше работает с ошибками (более понятные сообщения)

---

### Optional свойства

Свойство может быть необязательным:

```typescript
interface WeatherData {
  city: string;
  temperature: number;
  humidity?: number;        // Опционально
  windSpeed?: number;       // Опционально
}

const weather: WeatherData = {
  city: "Москва",
  temperature: 20
  // humidity и windSpeed можно не указывать
};
```

---

### Readonly свойства

Свойство нельзя изменить после создания:

```typescript
interface Config {
  readonly apiKey: string;
  readonly apiUrl: string;
  timeout: number;
}

const config: Config = {
  apiKey: "abc123",
  apiUrl: "https://api.weather.com",
  timeout: 5000
};

config.timeout = 10000;      // ✅ Можно изменить
config.apiKey = "new-key";   // ❌ Ошибка: Cannot assign to 'apiKey'
```

---

### Методы в интерфейсах

```typescript
interface Calculator {
  add(a: number, b: number): number;
  subtract(a: number, b: number): number;
}

const calc: Calculator = {
  add(a, b) {
    return a + b;
  },
  subtract(a, b) {
    return a - b;
  }
};
```

**Альтернативный синтаксис для методов:**

```typescript
interface Calculator {
  add: (a: number, b: number) => number;
  subtract: (a: number, b: number) => number;
}
```

## 🔤 Type Aliases

Type alias создаёт **псевдоним** для типа — можно описывать не только объекты, но и примитивы, union, tuple и т.д.

### Базовый синтаксис

```typescript
type User = {
  id: number;
  name: string;
  email: string;
};

const user: User = {
  id: 1,
  name: "Иван",
  email: "ivan@example.com"
};
```

**Преимущества:**
- ✅ Универсальность (можно описать любой тип)
- ✅ Union и intersection типы
- ✅ Tuple и примитивы
- ✅ Mapped и conditional типы (продвинутые фичи)

---

### Type aliases для union типов

```typescript
type Status = "idle" | "loading" | "success" | "error";
type ID = number | string;
type Response = SuccessResponse | ErrorResponse;
```

> 💡 **Важно:** Интерфейсы не могут представлять union типы — используй type alias.

---

### Type aliases для примитивов

```typescript
type Username = string;
type Temperature = number;
type IsActive = boolean;

let username: Username = "Иван";
let temp: Temperature = 20;
```

> 💡 Это полезно для семантики, но используй умеренно — избыток псевдонимов усложняет код.

---

## ⚖️ Различия между Interface и Type

| Возможность | Interface | Type Alias |
|-------------|-----------|------------|
| Описание объектов | ✅ | ✅ |
| Union типы | ❌ | ✅ |
| Tuple типы | ❌ | ✅ |
| Примитивы | ❌ | ✅ |
| Расширение (`extends`) | ✅ | ❌ (используй `&`) |
| Declaration Merging | ✅ | ❌ |
| Computed properties | ❌ | ✅ |
| Mapped Types | ❌ | ✅ |

---

### Пример: Union типы (только type)

```typescript
// ✅ Type alias
type Result = Success | Error;

// ❌ Interface не может
interface Result = Success | Error; // Ошибка синтаксиса
```

---

### Пример: Tuple типы (только type)

```typescript
// ✅ Type alias
type Coordinates = [number, number];

// ❌ Interface не может описать tuple напрямую
```

---

## 🔗 Расширение интерфейсов (extends)

Интерфейсы можно расширять, создавая иерархию типов:

```typescript
interface Animal {
  name: string;
  age: number;
}

interface Dog extends Animal {
  breed: string;
  bark(): void;
}

const dog: Dog = {
  name: "Шарик",
  age: 3,
  breed: "Лабрадор",
  bark() {
    console.log("Гав!");
  }
};
```

**Множественное наследование:**

```typescript
interface Flyable {
  fly(): void;
}

interface Swimmable {
  swim(): void;
}

interface Duck extends Flyable, Swimmable {
  quack(): void;
}
```

---

## 🔀 Пересечение типов (Intersection)

Type aliases используют `&` для объединения типов:

```typescript
type Person = {
  name: string;
  age: number;
};

type Employee = {
  employeeId: number;
  department: string;
};

type Worker = Person & Employee;

const worker: Worker = {
  name: "Иван",
  age: 30,
  employeeId: 12345,
  department: "IT"
};
```

**Interface vs Type для расширения:**

```typescript
// Interface
interface Dog extends Animal {
  breed: string;
}

// Type (эквивалент)
type Dog = Animal & {
  breed: string;
};
```

## 🔑 Индексные сигнатуры

Используются когда **ключи объекта неизвестны заранее**, но известен их тип.

### В интерфейсах

```typescript
interface Dictionary {
  [key: string]: string;
}

const translations: Dictionary = {
  hello: "привет",
  goodbye: "пока",
  thanks: "спасибо"
};

translations["welcome"] = "добро пожаловать"; // ✅
translations["count"] = 42; // ❌ Ошибка: Type 'number' is not assignable
```

---

### В type aliases

```typescript
type WeatherCache = {
  [city: string]: number;
};

const cache: WeatherCache = {
  "Москва": 20,
  "Казань": 18,
  "Санкт-Петербург": 15
};

cache["Новосибирск"] = 10; // ✅
```

---

### Комбинирование с известными свойствами

```typescript
interface User {
  id: number;
  name: string;
  [key: string]: any; // Дополнительные свойства
}

const user: User = {
  id: 1,
  name: "Иван",
  email: "ivan@example.com",  // Дополнительное свойство
  age: 25                      // Дополнительное свойство
};
```

> ⚠️ **Осторожно:** Индексные сигнатуры с `any` ослабляют типизацию. Используй конкретные типы где возможно.

---

## 📊 Когда использовать Interface vs Type

### Используй Interface когда:

✅ Описываешь структуру объектов <br/>
✅ Создаёшь публичные API (библиотеки) <br/>
✅ Нужно расширение через `extends` <br/>
✅ Работаешь с классами

**Пример:**
```typescript
interface ApiResponse {
  status: number;
  data: any;
}

interface SuccessResponse extends ApiResponse {
  data: WeatherData;
}
```

---

### Используй Type когда:

✅ Нужны Union или Intersection типы <br/>
✅ Описываешь примитивы, tuple, функции <br/>
✅ Используешь продвинутые типы (Mapped, Conditional) <br/>
✅ Создаёшь утилитарные типы

**Пример:**
```typescript
type Status = "idle" | "loading" | "success" | "error";
type Coordinates = [lat: number, lon: number];
type ApiResult = SuccessResponse | ErrorResponse;
```

## 🧩 Практические примеры

```typescript
interface BaseResponse {
  status: number;
  timestamp: number;
}

interface SuccessResponse extends BaseResponse {

interface ErrorResponse extends BaseResponse {
  error: string;

```

## 🔧 Практика 3.1.5.4: Индексные сигнатуры для кэша

Создадим типизированный кэш для хранения данных погоды.

### Шаг 0: Создай модуль кэша

Создай `src/modules/utils/cache.ts`:

```typescript
// src/modules/utils/cache.ts

type WeatherCache;

type CacheEntry;

const cache = {};

export function getCached(city) {
  const entry = cache[city];

  if (!entry) return null;

  const now = Date.now();
  const age = now - entry.timestamp;
  const maxAge = 10 * 60 * 1000;

  if (age > maxAge) {
    delete cache[city];
    return null;
  }

  return entry.data;
}

export function setCache(city, data) {
  cache[city] = {
    data,
    timestamp: Date.now()
  };
}

export function clearCache() {
  for (const key in cache) {
    delete cache[key];
  }
}

export function getCacheStats() {
  return Object.keys(cache).length;
}
```

---

### Задания

**1. Создай тип CacheEntry:**
   - `data` — данные погоды (WeatherData)
   - `timestamp` — время сохранения (число)

**2. Создай тип WeatherCache с индексной сигнатурой:**
   - Ключ — название города (строка)
   - Значение — запись кэша (CacheEntry)

**3. Типизируй переменную cache:**
   - Тип: `WeatherCache`
   - Начальное значение: пустой объект

**4. Типизируй функции:**
   - `getCached(city)` — принимает название города, возвращает данные погоды или null
   - `setCache(city, data)` — принимает город и данные погоды, ничего не возвращает
   - `clearCache()` — без параметров, ничего не возвращает

**5. Интегрируй кэш в API модуль:**

Открой `src/modules/api/weather.ts` и добавь логику кэширования:

   - **Импортируй функции кэша** в начало файла
   - **Перед запросом к API:**
     - Проверь, есть ли данные в кэше для этого города
     - Если есть — верни их сразу (тип уже `WeatherData`, сигнатура `fetchWeather` не меняется)
     - Добавь `console.log` для отладки: "✅ Кэш: данные для {город} получены из кэша"
   - **После успешного запроса к API:**
     - Сохрани полученные данные погоды в кэш
     - Добавь `console.log`: "🌐 API: данные для {город} сохранены в кэш"
   - **Важно:** Сохраняй в кэш только `WeatherData`, а не весь `CurrentWeather`

**6. Проверь работу кэша:**
   - Запроси погоду для одного города дважды
   - Первый запрос должен показать "🌐 API"
   - Второй запрос должен показать "✅ Кэш"

---

### Проверка базового кэша

```typescript
const mockData: WeatherData = {

```

```typescript
export function getCached(city: string): WeatherData | null {
  const entry = cache[city];
  if (!entry) return null;

  const now = Date.now();
  const age = now - entry.timestamp;
  const maxAge = 10 * 60 * 1000;

  if (age > maxAge) {
    delete cache[city];
    return null;
  }

  return entry.data;
}

export function setCache(city: string, data: WeatherData): void {
  cache[city] = {
    data,
    timestamp: Date.now()
  };
}
```

## ❌ Частые ошибки

### Ошибка 1: Попытка использовать interface для union

**Проблема:**
```typescript
interface Status = "idle" | "loading"; // ❌ Ошибка синтаксиса
```

**Решение:**
```typescript
type Status = "idle" | "loading"; // ✅
```

---

### Ошибка 2: Забыл optional (?) для необязательного свойства

**Проблема:**
```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const user: User = {
  id: 1,
  name: "Иван"
  // ❌ Ошибка: Property 'email' is missing
};
```

**Решение:**
```typescript
interface User {
  id: number;
  name: string;
  email?: string;
}

const user: User = {
  id: 1,
  name: "Иван"  // ✅ email опционален
};
```

---

### Ошибка 3: Изменение readonly свойства

**Проблема:**
```typescript
interface Config {
  readonly apiKey: string;
}

const config: Config = { apiKey: "abc123" };
config.apiKey = "new-key"; // ❌ Cannot assign to 'apiKey'
```

**Решение:**
```typescript
// Если нужно изменить — не используй readonly
interface Config {
  apiKey: string;
}

// Или создай новый объект
const newConfig: Config = { apiKey: "new-key" };
```

---

### Ошибка 4: Неправильная индексная сигнатура

**Проблема:**
```typescript
interface Cache {
  [key: string]: number;
}

const cache: Cache = {
  москва: 20,
  казань: "18" // ❌ Type 'string' is not assignable to type 'number'
};
```

**Решение:**
```typescript
interface Cache {
  [key: string]: number | string;
}

// Или используй конкретный тип
interface Cache {
  [key: string]: number;
}

const cache: Cache = {
  москва: 20,
  казань: 18  // ✅
};
```
