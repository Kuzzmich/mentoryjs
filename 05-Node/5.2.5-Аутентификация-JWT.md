# 5.2.5 Аутентификация (JWT)

Аутентификация — это проверка того, кто делает запрос. В REST API для этого используют токены: клиент получает токен при входе и отправляет его с каждым последующим запросом. Сервер проверяет токен и пускает или не пускает дальше. Здесь разберём полную цепочку: регистрация → вход → защищённые маршруты.

## 📖 Хеширование паролей с bcrypt

Пароли нельзя хранить в открытом виде. Даже если база данных утечёт, злоумышленник не должен получить пароли пользователей. `bcrypt` — алгоритм хеширования, специально разработанный для паролей: медленный намеренно, чтобы перебор был дорогим.

```bash
npm install bcrypt
npm install -D @types/bcrypt
```

```typescript
import bcrypt from "bcrypt";

const SALT_ROUNDS = 10; // сложность хеширования

// Хеширование при регистрации:
const hash = await bcrypt.hash("userPassword123", SALT_ROUNDS);
// "$2b$10$..." — строка фиксированной длины, безопасно хранить в БД

// Проверка при входе:
const isValid = await bcrypt.compare("userPassword123", hash);
// true — пароль совпадает
// false — не совпадает
```

`bcrypt.compare` не расшифровывает хеш — она хеширует введённый пароль тем же алгоритмом и сравнивает результаты. Оригинальный пароль никогда не хранится.

## 📖 Что такое JWT

JWT (JSON Web Token) — строка из трёх частей, разделённых точкой: `header.payload.signature`.

- **Header** — алгоритм подписи (`HS256`)
- **Payload** — данные: `id` пользователя, `email`, время истечения
- **Signature** — подпись, которую сервер проверяет своим секретным ключом

```
eyJhbGciOiJIUzI1NiJ9.eyJpZCI6IjEyMyIsImVtYWlsIjoiQG1haWwifQ.xyzSignature
```

Токен не зашифрован — payload можно декодировать без ключа. Но без ключа нельзя создать валидную подпись. Поэтому в payload не кладут пароли и другие секреты — только то, что можно прочитать: id, email, роль.

## 📖 Генерация и верификация токенов

```bash
npm install jsonwebtoken
npm install -D @types/jsonwebtoken
```

```typescript
import jwt from "jsonwebtoken";

const JWT_SECRET = process.env.JWT_SECRET!;

// Тип данных в токене
interface JwtPayload {
  id: string;
  email: string;
}

// Генерация токена при входе:
function generateToken(payload: JwtPayload): string {
  return jwt.sign(payload, JWT_SECRET, { expiresIn: "7d" });
}

// Верификация токена:
function verifyToken(token: string): JwtPayload {
  return jwt.verify(token, JWT_SECRET) as JwtPayload;
}
```

`expiresIn: "7d"` — токен действует 7 дней. После этого `jwt.verify` выбросит `TokenExpiredError`. Можно использовать `"1h"`, `"30m"`, `"7d"` и другие форматы.

## 📖 Хранилище пользователей

В этом разделе пользователи хранятся в памяти — без базы данных. Это упрощает код и позволяет сосредоточиться на логике авторизации. В реальном проекте вместо массива будет MongoDB или PostgreSQL.

```typescript
// src/services/auth.service.ts
import bcrypt from "bcrypt";
import jwt from "jsonwebtoken";
import { config } from "../config";
import { AppError } from "../middleware/AppError";

interface User {
  id: string;
  email: string;
  name: string;
  passwordHash: string;
}

// Хранилище в памяти
const users: User[] = [];

export interface JwtPayload {
  id: string;
  email: string;
}

export async function register(
  email: string,
  password: string,
  name: string
): Promise<{ token: string }> {
  const existing = users.find((u) => u.email === email);
  if (existing) {
    throw new AppError("Пользователь с таким email уже существует", 409);
  }

  const passwordHash = await bcrypt.hash(password, 10);
  const user: User = {
    id: crypto.randomUUID(),
    email,
    name,
    passwordHash,
  };

  users.push(user);

  const token = jwt.sign({ id: user.id, email: user.email }, config.jwtSecret, {
    expiresIn: "7d",
  });

  return { token };
}

export async function login(
  email: string,
  password: string
): Promise<{ token: string }> {
  const user = users.find((u) => u.email === email);
  if (!user) {
    throw new AppError("Неверный email или пароль", 401);
  }

  const isValid = await bcrypt.compare(password, user.passwordHash);
  if (!isValid) {
    throw new AppError("Неверный email или пароль", 401);
  }

  const token = jwt.sign({ id: user.id, email: user.email }, config.jwtSecret, {
    expiresIn: "7d",
  });

  return { token };
}
```

Обрати внимание: при неверном пароле и при несуществующем email возвращается одна и та же ошибка. Это намеренно — раскрывать, существует ли email в системе, нежелательно.

## 📖 authMiddleware и расширение типа Request

`authMiddleware` проверяет токен из заголовка `Authorization: Bearer <token>` и кладёт данные пользователя в `req.user`. Чтобы TypeScript об этом знал, расширяют тип `Request`:

```typescript
// src/types/express.d.ts
import { JwtPayload } from "../services/auth.service";

declare global {
  namespace Express {
    interface Request {
      user?: JwtPayload;
    }
  }
}
```

Этот файл не нужно импортировать — TypeScript подхватит его автоматически, так как он находится в `src/`.

```typescript
// src/middleware/auth.middleware.ts
import { Request, Response, NextFunction } from "express";
import jwt from "jsonwebtoken";
import { config } from "../config";
import { JwtPayload } from "../services/auth.service";

export function authMiddleware(
  req: Request,
  res: Response,
  next: NextFunction
): void {
  const authHeader = req.headers.authorization;

  if (!authHeader || !authHeader.startsWith("Bearer ")) {
    res.status(401).json({ error: "Требуется авторизация" });
    return;
  }

  const token = authHeader.slice(7); // убираем "Bearer "

  try {
    const payload = jwt.verify(token, config.jwtSecret) as JwtPayload;
    req.user = payload;
    next();
  } catch {
    res.status(401).json({ error: "Токен недействителен или истёк" });
  }
}
```

## 📖 Защита маршрутов

`authMiddleware` применяют к маршрутам, которые требуют авторизации:

```typescript
// src/routes/favorites.router.ts
import { Router } from "express";
import { authMiddleware } from "../middleware/auth.middleware";
import { asyncHandler } from "../middleware/asyncHandler";
import { getAll, addCity } from "../controllers/favorites.controller";

const router = Router();

router.use(authMiddleware); // защита всего роутера

router.get("/", asyncHandler(getAll));
router.post("/", asyncHandler(addCity));

export default router;
```

В контроллере теперь доступен `req.user`:

```typescript
export async function getAll(req: Request, res: Response): Promise<void> {
  const userId = req.user!.id; // TypeScript знает, что user существует
  const favorites = getFavoritesByUser(userId);
  res.json(favorites);
}
```

## 📖 Маршруты регистрации и входа

```typescript
// src/routes/auth.router.ts
import { Router } from "express";
import { validate } from "../middleware/validate";
import { asyncHandler } from "../middleware/asyncHandler";
import { RegisterSchema, LoginSchema } from "../schemas/auth.schemas";
import * as authController from "../controllers/auth.controller";

const router = Router();

router.post("/register", validate(RegisterSchema), asyncHandler(authController.register));
router.post("/login", validate(LoginSchema), asyncHandler(authController.login));

export default router;
```

```typescript
// src/controllers/auth.controller.ts
import { Request, Response } from "express";
import * as authService from "../services/auth.service";
import { RegisterBody, LoginBody } from "../schemas/auth.schemas";

export async function register(
  req: Request<{}, {}, RegisterBody>,
  res: Response
): Promise<void> {
  const { email, password, name } = req.body;
  const result = await authService.register(email, password, name);
  res.status(201).json(result);
}

export async function login(
  req: Request<{}, {}, LoginBody>,
  res: Response
): Promise<void> {
  const { email, password } = req.body;
  const result = await authService.login(email, password);
  res.json(result);
}
```

В `config.ts` добавь `jwtSecret`:

```typescript
export const config = {
  port: Number(process.env.PORT) || 3000,
  nodeEnv: process.env.NODE_ENV || "development",
  weatherApiKey: process.env.WEATHER_API_KEY || "",
  jwtSecret: process.env.JWT_SECRET || "dev-secret-change-in-production",
};
```

В `.env` и `.env.example`:

```bash
JWT_SECRET=your-very-long-random-secret-key
```

## 🎯 Финальное задание

Сделай избранное персональным: каждый пользователь видит только свои города.

1. В `favorites.service.ts` измени хранилище на `Map<string, string[]>` — ключ `userId`, значение — массив городов
2. Обнови `getFavorites`, `addFavorite`, `removeFavorite` — принимают `userId` первым параметром
3. В `favorites.controller.ts` передавай `req.user!.id` в сервис

После этого два разных пользователя, добавив города, увидят только свои списки.
