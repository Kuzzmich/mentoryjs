# 5.1.1 Что такое Node.js

Node.js — это среда выполнения JavaScript вне браузера. Один и тот же язык, другое окружение: нет DOM и `window`, но есть доступ к файловой системе, сети и процессам операционной системы.

## 📖 Как работает Node.js

Node.js использует движок V8 (тот же, что в Chrome) и добавляет к нему библиотеку libuv — она отвечает за ввод-вывод без блокировки. Когда Node.js читает файл или делает сетевой запрос, он не ждёт результата, а продолжает выполнять другой код. Результат придёт в коллбэке, Promise или через `await`.

Именно поэтому Node.js хорошо справляется с большим количеством одновременных запросов при небольшом потреблении памяти.

---

## 📖 Чем серверная среда отличается от браузерной

В браузере есть `window`, `document`, `localStorage`, `navigator`. В Node.js их нет — и обращение к ним вызовет ошибку `ReferenceError`.

Вместо них в Node.js есть другие глобальные объекты:

```typescript
// Информация о текущем процессе
console.log(process.version);      // версия Node.js
console.log(process.platform);     // "linux", "darwin", "win32"
console.log(process.env.HOME);     // переменные окружения
console.log(process.argv);         // аргументы командной строки

// Путь к текущему файлу и папке (CommonJS)
console.log(__filename); // /home/user/project/src/index.ts
console.log(__dirname);  // /home/user/project/src

// В ESM-модулях используют import.meta.url вместо __dirname
```

`process` — глобальный объект, импортировать не нужно. Он доступен в любом файле.

## 📖 Модуль fs — работа с файлами

`fs` (file system) — встроенный модуль для чтения, записи и управления файлами.

```typescript
import { readFile, writeFile, readFileSync } from "fs";
import { promisify } from "util";

// Синхронное чтение — блокирует выполнение до завершения
const data = readFileSync("./cities.json", "utf-8");
const cities = JSON.parse(data);

// Асинхронное чтение через промисы — предпочтительный способ
import { readFile as readFileAsync, writeFile as writeFileAsync } from "fs/promises";

async function loadCities(): Promise<string[]> {
  const raw = await readFileAsync("./cities.json", "utf-8");
  return JSON.parse(raw);
}

async function saveCities(cities: string[]): Promise<void> {
  await writeFileAsync("./cities.json", JSON.stringify(cities, null, 2), "utf-8");
}
```

`fs/promises` — версия модуля с Promise-интерфейсом. Используй её везде, где нужна асинхронность. `readFileSync` подходит только для скриптов при старте приложения, когда блокировка не критична.

## 📖 Модуль path — работа с путями

Пути к файлам на Windows и Linux/macOS различаются разделителем (`\` vs `/`). `path` позволяет строить пути кроссплатформенно.

```typescript
import path from "path";

// path.join — соединяет сегменты пути правильным разделителем
const filePath = path.join(__dirname, "data", "cities.json");
// Linux: /project/src/data/cities.json
// Windows: \project\src\data\cities.json

// path.resolve — возвращает абсолютный путь
const absPath = path.resolve("cities.json");
// /текущая/рабочая/директория/cities.json

// Разбор пути
const p = "/project/src/weather.service.ts";
path.dirname(p);   // "/project/src"
path.basename(p);  // "weather.service.ts"
path.extname(p);   // ".ts"
```

Всегда используй `path.join` вместо конкатенации строк — это избавит от проблем с разделителями на разных ОС.

## 📖 Запуск TypeScript через tsx

`tsx` — инструмент для запуска TypeScript-файлов напрямую, без предварительной компиляции. Это удобно при разработке.

```bash
npm install -D tsx
```

```json
// package.json
{
  "scripts": {
    "dev": "tsx src/index.ts",
    "dev:watch": "tsx watch src/index.ts"
  }
}
```

```bash
npm run dev          # запустить один раз
npm run dev:watch    # перезапускать при изменениях
```

`tsx watch` — аналог `nodemon`, но без дополнительных зависимостей.

async function loadCities(): Promise<string[]> {
  try {
    const raw = await readFile(CITIES_FILE, "utf-8");
    return JSON.parse(raw);

<summary>💡 Подсказка 2: saveCity</summary>

  }
}
