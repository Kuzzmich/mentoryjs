# 4.2.1 Composables

Composable — это функция с префиксом `use`, которая инкапсулирует реактивное состояние и логику. Вместо того чтобы копировать один и тот же код в каждый компонент — ты выносишь его в composable и переиспользуешь.

---

## 🎯 Что ты узнаешь

- Что такое composable и чем он отличается от обычной функции
- Как создавать composables и какие правила нужно соблюдать
- `useWeather()` — вынести логику загрузки погоды
- `useLocalStorage()` — реактивная обёртка над localStorage
- `useDebounce()` — переиспользуемый debounce

---

## 📖 Зачем нужны composables

Посмотри на типичный компонент после нескольких разделов:

```vue
<script setup lang="ts">
// Логика погоды
const weatherData = ref(null);
const isLoading = ref(false);
const error = ref(null);
async function fetchWeather(city) { ... }
onMounted(() => fetchWeather(props.city));
watch(() => props.city, fetchWeather);
let refreshInterval = null;
onMounted(() => { refreshInterval = setInterval(...) });
onUnmounted(() => clearInterval(refreshInterval));

// Логика единиц измерения
const unit = ref("celsius");
function toggleUnit() { ... }
const displayTemp = computed(() => ...);

// Логика избранного
const favorites = ref([]);
const isFavorite = computed(() => ...);
function toggleFavorite() { ... }
</script>
```

Компонент делает слишком много. Если та же логика нужна в другом компоненте — копируешь. Composables решают это:

```vue
<script setup lang="ts">
// Теперь каждый блок — отдельная переиспользуемая функция
const { weatherData, isLoading, error } = useWeather(props.city);
const { displayTemp, unit, toggleUnit } = useTemperatureUnit(weatherData);
const { isFavorite, toggleFavorite } = useFavorites(weatherData);
</script>
```

---

## 📖 Правила composables

**1. Префикс `use`** — обязательное соглашение. `useWeather`, `useLocalStorage`, `useDebounce`. Это сигнал что функция содержит реактивное состояние.

**2. Вызов только в `setup`** — composables используют `ref`, `computed`, `onMounted` и другие API, которые работают только в контексте компонента:

```typescript
// ✅ Правильно — вызов в script setup
const { data } = useWeather("Москва");

// ❌ Неправильно — вызов вне компонента
function someHelper() {
  const { data } = useWeather("Москва"); // onMounted не сработает
}
```

**3. Возвращай `ref`, а не `.value`** — чтобы деструктуризация не сломала реактивность:

```typescript
// ✅ Возвращаем ref — деструктуризация сохраняет реактивность
export function useCounter() {
  const count = ref(0);
  return { count }; // не count.value
}

// В компоненте:
const { count } = useCounter(); // count — реактивный ref
```

**4. Файлы — в `src/composables/`** — стандартная директория.

## 📖 Пример: useWeather

```typescript
// src/composables/useWeather.ts
import { ref, watch, onMounted, onUnmounted } from "vue";
import type { Ref } from "vue";
import type { WeatherData } from "../types/weather";

export function useWeather(city: Ref<string> | string) {
  const weatherData = ref<WeatherData | null>(null);
  const isLoading = ref(false);
  const error = ref<string | null>(null);
  const lastUpdated = ref<Date | null>(null);

  const API_KEY = import.meta.env.VITE_WEATHER_API_KEY;

  async function fetchWeather(cityName: string) {
    isLoading.value = true;
    error.value = null;
    try {
      const response = await fetch(
        `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(cityName)}&appid=${API_KEY}&units=metric&lang=ru`
      );
      if (!response.ok) throw new Error("Город не найден");
      const data = await response.json();
      weatherData.value = {
        city: data.name,
        country: data.sys.country,
        temperature: Math.round(data.main.temp),
        feelsLike: Math.round(data.main.feels_like),
        humidity: data.main.humidity,
        description: data.weather[0].description,
        icon: data.weather[0].icon,
        timestamp: Date.now(),
      };
      lastUpdated.value = new Date();
    } catch (e) {
      error.value = e instanceof Error ? e.message : "Ошибка загрузки";
    } finally {
      isLoading.value = false;
    }
  }

  // Принимаем и ref и строку
  const cityRef = typeof city === "string" ? ref(city) : city;

  onMounted(() => fetchWeather(cityRef.value));
  watch(cityRef, fetchWeather);

  let refreshInterval: ReturnType<typeof setInterval> | null = null;
  onMounted(() => {
    refreshInterval = setInterval(() => fetchWeather(cityRef.value), 30_000);
  });
  onUnmounted(() => {
    if (refreshInterval) clearInterval(refreshInterval);
  });

  return { weatherData, isLoading, error, lastUpdated, fetchWeather };
}
```

Использование в компоненте стало простым:

```vue
<script setup lang="ts">
import { toRef } from "vue";
import { useWeather } from "../composables/useWeather";

const props = defineProps<{ city: string }>();

// toRef сохраняет реактивность — composable будет реагировать на смену props.city
const { weatherData, isLoading, error, lastUpdated } = useWeather(toRef(props, "city"));
</script>
```

## 📖 Пример: useLocalStorage

Composable не обязан использовать хуки жизненного цикла — он может просто возвращать реактивное состояние с удобным API:

```typescript
// src/composables/useLocalStorage.ts
import { ref, watch } from "vue";

export function useLocalStorage<T>(key: string, defaultValue: T) {
  const stored = localStorage.getItem(key);
  const data = ref<T>(stored ? JSON.parse(stored) : defaultValue);

  // Автоматически сохраняем в localStorage при каждом изменении
  watch(data, (newValue) => {
    localStorage.setItem(key, JSON.stringify(newValue));
  }, { deep: true });

  return data;
}
```

Использование:

```typescript
// Вместо ручного watch + localStorage в App.vue:
const cities = useLocalStorage("cities", ["Москва", "Санкт-Петербург", "Казань"]);
// cities — реактивный ref, автоматически синхронизируется с localStorage
```

```typescript
import { ref } from "vue";

export function useDebounce<T extends (...args: unknown[]) => void>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout> | null = null;

  return (...args: Parameters<T>) => {
    if (timer !== null) clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}
```
