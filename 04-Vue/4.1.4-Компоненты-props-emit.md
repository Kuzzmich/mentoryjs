# 4.1.4 Компоненты: props и emit

Vue-приложение — это дерево компонентов. Данные всегда текут сверху вниз через props, а события — снизу вверх через emit. Это правило называют «однонаправленным потоком данных», и оно делает приложение предсказуемым.

---

## 🎯 Что ты узнаешь

- Как передавать данные в компонент через `defineProps<T>()`
- Как компонент сообщает родителю о событиях через `defineEmits<T>()`
- Что такое однонаправленный поток данных и зачем он нужен
- Как типизировать props и emit с TypeScript

---

## 📖 Props — данные сверху вниз

Props — это параметры компонента. Родитель передаёт данные, дочерний компонент их получает и использует, но **не изменяет**.

### defineProps с TypeScript

```vue
<!-- src/components/WeatherCard.vue -->
<script setup lang="ts">
import type { WeatherData } from "../types/weather";

const props = defineProps<{
  city: string;
  weatherData: WeatherData;
  isLoading?: boolean;  // ? — необязательный prop
}>();
</script>

<template>
  <div class="weather-card">
    <h2>{{ props.city }}</h2>
    <p v-if="props.isLoading">Загрузка...</p>
    <p v-else>{{ props.weatherData.temperature }}°C</p>
  </div>
</template>
```

Использование в родителе:

```vue
<!-- src/App.vue -->
<template>
  <WeatherCard
    :city="selectedCity"
    :weather-data="currentWeather"
    :is-loading="isLoading"
  />
</template>
```

> В шаблоне camelCase превращается в kebab-case: `weatherData` → `:weather-data`. Vue принимает оба варианта.

### Значения по умолчанию

Для необязательных props задай дефолтные значения через `withDefaults`:

```typescript
const props = withDefaults(
  defineProps<{
    city: string;
    isLoading?: boolean;
    unit?: "celsius" | "fahrenheit";
  }>(),
  {
    isLoading: false,
    unit: "celsius",
  }
);
```

### Чего нельзя делать с props

Props — только для чтения. Нельзя присваивать им значения напрямую:

```typescript
// ❌ Нарушает однонаправленный поток — Vue выдаст предупреждение
props.city = "Казань";

// ✅ Если нужно изменяемое значение — скопируй в ref
const localCity = ref(props.city);
```

## 📖 Emit — события снизу вверх

Когда дочерний компонент хочет сообщить родителю о действии пользователя — он генерирует событие через `emit`. Родитель подписывается через `@`.

### defineEmits с TypeScript

```vue
<!-- src/components/WeatherCard.vue -->
<script setup lang="ts">
const emit = defineEmits<{
  refresh: [city: string];           // событие без возврата
  unitChange: [unit: "celsius" | "fahrenheit"];
  favorite: [city: string, isFav: boolean];
}>();

function handleRefresh() {
  emit("refresh", props.city);
}

function handleUnitToggle() {
  const newUnit = props.unit === "celsius" ? "fahrenheit" : "celsius";
  emit("unitChange", newUnit);
}
</script>
```

Подписка в родителе:

```vue
<!-- src/App.vue -->
<template>
  <WeatherCard
    :city="selectedCity"
    :weather-data="currentWeather"
    @refresh="loadWeather"
    @unit-change="handleUnitChange"
    @favorite="handleFavorite"
  />
</template>

<script setup lang="ts">
function loadWeather(city: string) {
  console.log("Обновляем:", city);
}

function handleUnitChange(unit: "celsius" | "fahrenheit") {
  selectedUnit.value = unit;
}

function handleFavorite(city: string, isFav: boolean) {
  console.log(city, isFav ? "добавлен" : "удалён");
}
</script>
```

## 📖 Однонаправленный поток данных

Правило простое: **данные идут вниз, события — вверх**.

```
App.vue (selectedCity, weatherData)
    │  props: city, weatherData
    ▼
WeatherCard.vue
    │  emit: refresh, unitChange
    ▲
App.vue обрабатывает события, обновляет данные
```

Почему это важно: если каждый компонент мог бы менять данные произвольно, отследить откуда пришло изменение стало бы невозможно. Однонаправленный поток делает баги предсказуемыми.

Пример нарушения и правильного подхода:

```vue
<!-- ❌ Плохо: дочерний компонент меняет свой prop -->
<script setup>
const props = defineProps<{ city: string }>();
props.city = "Казань"; // нарушение
</script>

<!-- ✅ Хорошо: дочерний компонент эмитит событие -->
<script setup>
const props = defineProps<{ city: string }>();
const emit = defineEmits<{ cityChange: [city: string] }>();

function changeCity() {
  emit("cityChange", "Казань"); // родитель решает что делать
}
</script>
```

---

## 📖 Как распределять ответственность

Хорошее правило: компонент владеет теми данными, которые он создаёт. Если данные нужны нескольким компонентам — подними их в общего родителя.

| Вопрос | Ответ |
|---|---|
| Кто хранит `weatherData`? | `App.vue` — он передаёт в несколько мест |
| Кто хранит `unit` (°C/°F)? | `WeatherCard` — это его внутреннее состояние |
| Кто хранит `favorites`? | `App.vue` — список нужен нескольким компонентам |
| Как `WeatherCard` запрашивает обновление? | Через `emit("refresh", city)` |
